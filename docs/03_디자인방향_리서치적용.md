# THESIS 디자인 방향 — 리서치 적용 (다크모드 우선)

> 본 문서는 2026년 기준 핀테크·투자 SaaS·데이터 대시보드 리서치를 THESIS에 적용한 **실행 가능한 디자인 방향**이다.
> 모든 권장에는 근거/출처 URL을 단다. 추측 금지 — 검증된 출처와 검증된 코드(`frontend/app|components|lib/*`, `backend/app/*`)만 인용한다.
> **docs/01(기능명세)·02(페이지구성)와 모순 없이 보강**한다. 특히 02 §6-9의 `.dark` 스케치, 색 토큰 규칙(amber 기조·rose 강경계만), Top-Down 3중 강제(사이트맵=네비=섹션 순서), 표현 금지어 사전을 그대로 잇는다.
> 코드 현황 전제(02·01에서 확인): `globals.css`는 `:root` 라이트 토큰만 존재, `.dark` 블록·`tailwind darkMode` 설정·Framer Motion 미설치, `app/page.tsx`는 즉시 redirect(랜딩 없음), 대시보드는 `max-w-3xl` 단일 컬럼에 `RegimeSignalCard`+`AllocationPanel` 2개뿐.

## 1. 요약 — 무엇을, 왜

### 1-1. 문제
- **허전함**: 화면이 대시보드 하나뿐이고 카드 2개라 "이게 끝?" 느낌이 구조적으로 불가피.
- **정적임**: 데이터가 살아있다는 신호가 없어(신선도 칩·미세 전환 부재) 죽은 화면처럼 보임.
- **첫인상 없음**: `app/page.tsx`가 즉시 redirect → 로그인 전 랜딩이 없어 첫인상·SEO·전환 진입점을 통째로 버림.
- **테마**: 다크/라이트 미구현. 이번에 **다크모드 기본**으로 결정.

### 1-2. 해결 방향(우선순위 순)
1. **다크모드 토큰 시스템 구축** — 모든 시각·인터랙션 작업의 *선행 조건*. 다크 기본은 2026 핀테크의 프리미엄/신뢰 시그널(Mercury·Stripe·Brex). 출처: <https://www.blendb2b.com/blog/best-fintech-website-examples>
2. **로그인 전 랜딩 신설** — 제품 X-ray 히어로 + 국면 결론 미리보기 + 가격 투명. 핀테크 랜딩 전환율 5~8%. 출처: <https://designrevision.com/blog/fintech-saas-landing-pages>
3. **대시보드를 '결론 섹션 4~6개 스택 + 점진적 공개'로 재구성** — 풍부함은 데이터 밀도가 아니라 내러티브 섹션+요약→펼침. 출처: <https://www.uxpin.com/studio/blog/dashboard-design-principles/>
4. **상시 신선도 칩** — `cache_status`를 예외 배너에서 상시 Live/Stale 칩으로 격상. 정적 느낌 제거의 최저비용 수단. 출처: <https://www.smashingmagazine.com/2025/09/ux-strategies-real-time-dashboards/>
5. **절제된 인터랙션** — Framer Motion(LazyMotion+`m`+strict, 4.6KB) + NumberFlow(결론 수치만) + reduced-motion 기본 존중. 출처: <https://motion.dev/docs/react-reduce-bundle-size>

### 1-3. 절대 기준(위반 아이디어는 배제)
THESIS 8대 설계철학(01 §1-5). 특히 ①"지금 사세요" 금지 ③결론 먼저·raw 숨김 ⑤실시간/신선도 ⑥과신 방지 ⑦결론 무료/근거 유료. **트레이딩 도파민(가격 깜빡임·카운트다운·FOMO)·다크패턴은 전면 금지.** 모션 과용은 측정된 손해(이탈 +40%, 폼 전환 -50%). 출처: <https://www.brad-holmes.co.uk/web-performance-ux/why-most-scroll-animations-miss-what-apple-gets-right/>

## 2. 다크모드 우선 색 시스템

### 2-1. 핵심 원칙(근거)
| 원칙 | 권장 | 근거 |
|------|------|------|
| 순흑·순백 금지 | 배경 L≈9~13% 짙은 청회색, 본문 L≈90~93% 오프화이트 | halation/진동·눈피로. Material 권장 #121212. <https://atmos.style/blog/dark-mode-ui-best-practices> |

| 라이트 단순 반전 금지 | 다크 전용 변형(채도 약 15~20%p↓·명도 소폭↑) | 채도 높은 색은 다크 배경에서 시각 진동. = '패닉 레드 벽' 회피의 기술적 본질. <https://atmos.style/blog/dark-mode-ui-best-practices> |

| 색만으로 의미 전달 금지 | 상승/하락·국면에 ↑↓·+/-·라벨·수치 병기 | WCAG 1.4.1. 적록은 deuteranopia에 뭉개짐. <https://rgblind.com/blog/color-blindness-friendly-chart-colors> |
| 대비 재계산 | 본문 4.5:1, 비텍스트(차트/보더/아이콘) 3:1을 다크에서 전부 재산출 | WCAG 1.4.11. <https://www.digitala11y.com/understanding-sc-1-4-11-non-text-contrast/> |

### 2-2. CSS 변수 권장값
> 02 §6-9의 `.dark` 스케치를 기반으로, surface elevation(표면 명도 단계)과 onEmphasis(밝은 솔리드 위 어두운 텍스트), 미등록 토큰(`--regime-off-strong`, `-muted`/`-emphasis`/`-foreground`)을 마저 채운다. HSL 표기는 기존 `globals.css` 컨벤션(공백 구분, `hsl(var(...))` 래핑) 유지.

**다크(기본) — `.dark` 블록**
```css
.dark {
  /* 표면 elevation: 위로 갈수록 밝게 (그림자 대신 명도로 깊이) */
  --background: 220 24% 9%;       /* ≈#11151C 짙은 청회색, 순흑 금지 */
  --surface-1: 220 22% 12%;       /* 카드 base (+3%p) */
  --surface-2: 220 20% 15%;       /* hover/팝오버 (+3%p) */
  --card: 220 22% 12%;
  --card-foreground: 210 16% 92%; /* 오프화이트, 순백 금지 */
  --foreground: 210 16% 92%;
  --muted: 220 16% 18%;
  --muted-foreground: 215 14% 64%; /* 본문 대비 ≥4.5:1 확인 */
  --border: 220 16% 22%;           /* 미세 보더(저대비) — elevation과 병행 */

  /* regime: 라이트 반전 아님 — 다크 전용 채도↓·명도↑ 변형 */
  --regime-on: 152 48% 52%;            /* emerald 톤다운 */
  --regime-on-muted: 152 30% 18%;      /* 카드 배경 채움(옅음) */
  --regime-neutral: 215 14% 64%;       /* slate */
  --regime-off: 38 80% 56%;            /* amber 기조 유지 */
  --regime-off-muted: 38 45% 16%;
  --regime-off-strong: 350 62% 60%;    /* rose: 강경계만, 면적 최소 */
  --regime-off-strong-muted: 350 40% 18%;

  /* 밝은 솔리드(amber 등) 위 텍스트는 어둡게 반전 */
  --on-emphasis: 220 24% 9%;
}
```

**라이트 — `:root` (기존 유지 + 누락 토큰 추가)**
```css
:root {
  --background: 0 0% 100%;
  --foreground: 222 47% 11%;
  --surface-1: 0 0% 100%;
  --surface-2: 210 40% 98%;
  --card: 0 0% 100%;
  --card-foreground: 222 47% 11%;
  --muted: 210 40% 96%;
  --muted-foreground: 215 16% 47%;
  --border: 214 32% 91%;
  --radius: 0.75rem;

  --regime-on: 152 60% 40%;             /* emerald (기존) */
  --regime-on-muted: 152 50% 94%;
  --regime-neutral: 215 16% 47%;        /* slate (기존) */
  --regime-off: 32 95% 44%;             /* amber (기존) */
  --regime-off-muted: 38 92% 94%;
  --regime-off-strong: 350 78% 50%;     /* rose (기존) */
  --regime-off-strong-muted: 350 80% 96%;
  --on-emphasis: 0 0% 100%;
}
```

### 2-3. 적용 작업(코드)
- `tailwind.config.ts`에 `darkMode: "class"` 추가, `colors`에 미등록 토큰(`regime-off-strong`, `*-muted`, `surface-1/2`, `on-emphasis`) 등록. **현재 `regime-off-strong`는 CSS에만 있고 tailwind에 미등록**.
- 테마 토글: 시스템 우선(`prefers-color-scheme`) + 수동(localStorage). **단, 이번 결정은 다크 기본** → SSR 첫 페인트부터 `<html class="dark">`가 적용되도록 `layout.tsx`에 깜빡임 방지 인라인 스크립트(no-flash). 토글 위치는 헤더 + `/settings`(02 §4-12).
- `RegimeSignalCard`의 인라인 `style={{ backgroundColor: style.badgeBg }}`(lib/regime `REGIME_STYLES`)를 **CSS 변수 참조로 전환** → 테마 전환이 깨지지 않게. `AllocationDonut`의 `SLICE_COLORS`는 이미 `hsl(var(--regime-*))`라 자동 대응(코드 확인).
- 도넛 세그먼트 사이 **배경색 gap**(stroke=배경색, 1~2px)으로 색약 대비 보강. 현재 `stroke="none"`(코드 확인) → `stroke="hsl(var(--background))"`.

### 2-4. 대비·접근성 체크리스트(다크에서 재계산 필수)
- [ ] 본문 텍스트 `--foreground`/`--muted-foreground` vs `--background`/`--card` ≥ 4.5:1
- [ ] 국면 배지·도넛 세그먼트·보더·아이콘(비텍스트) ≥ 3:1 (WCAG 1.4.11)
- [ ] amber 등 밝은 솔리드 위 텍스트는 `--on-emphasis`(어두운 색) 사용
- [ ] confidence 막대: 색 + sr-only 텍스트(이미 코드에 있음) + 막대 수 중복 인코딩
- [ ] rose는 작은 배지/테두리/아이콘에만, 대면적 배경 채움 금지(02 §6-1)

## 4. 대시보드를 풍부하게

> 풍부함 = 데이터 밀도가 아니라 **'얇은 결론 섹션' 스택 + 요약→펼침(progressive disclosure)**. 입문자는 결론만, 중급자는 펼쳐 근거를. 출처: <https://www.uxpin.com/studio/blog/dashboard-design-principles/>
> 02 §6-0 Top-Down 순서·게이팅·표현 원칙을 그대로 따른다. 카드 총수 6~7개 이하 상한, `max-w-3xl` 단일 컬럼 유지(데스크톱만 2열 grid 군집화).

| 6 | **차분한 넛지 카드('오늘 살펴볼 한 가지')** | "원달러가 주의 구간에 들어왔습니다 — 환노출 자산 비중을 점검해 볼 시점" | Free | **행동 지시 금지** — '점검·검토' 넛지만(철학1·6) | Eleken one actionable insight 변형. <https://www.eleken.co/blog-posts/fintech-ux-best-practices> |

### 4-2. 점진적 공개(progressive disclosure)
- 각 섹션 = **한 줄 결론(기본 표시) + 펼침(근거, Pro 게이팅)**. 기존 `EvidenceLocked` 컴포넌트를 섹션마다 재사용.
- 펼침 인터랙션은 `grid-template-rows` 또는 Motion `layout` 트랜지션 250ms ease-out(레이아웃 jank 회피). 출처: <https://www.uxpin.com/studio/blog/dashboard-design-principles/>
- **철학3 보강(01 §4)**: Pro evidence도 기본 접힘 + "근거 수치 보기" 토글(Pro 입문자 면제 안 함).

### 4-3. 신뢰형 빈 상태
- `AllocationPanel` 입력 전(result=null): "투자 기간·성향을 고르고 근거를 살펴보세요"(02 §6-7).
- 포트폴리오 0종목: "보유 종목을 추가해 시작" + 다음 단계 CTA. **차분·안내 톤**(축하 애니메이션·배지 남발 금지 — 투자 맥락에서 과신 유발). 출처: <https://www.eleken.co/blog-posts/empty-state-ux>

### 4-4. 시각 위계 3단
국면 결론(대형·시맨틱색) > 자산배분 도넛(중형) > 지표 상세(소형·Pro 잠금). 데스크톱은 매크로 보드·자산배분을 2열 grid로 묶어 '여백 있는 풍부함'. 출처: <https://www.uxpin.com/studio/blog/dashboard-design-principles/>

## 5. 신뢰 기반 리텐션

> 성공지표를 'DAU/세션수/거래수'가 아니라 **'국면 이해도·분산 점검 빈도·변동성 구간 자제율'**(Vanguard ACE)로 정의. 모든 신규 인게이지먼트 기능은 'ACE 체크리스트(투자자 결과 개선? 자율성 보존?)'를 통과. 출처: <https://corporate.vanguard.com/content/dam/corp/research/pdf/principles_for_behavioral_design_nudging_for_better_investor_outcomes.pdf>

### 5-1. 채택안
| 전술 | 구현 | 철학 정합 | 근거 |
|------|------|-----------|------|
| **최초 로그인 온보딩(3스텝)** | 02 §3-2 그대로(국면 읽는 법→감정 슬라이더→결론무료/근거유료), localStorage 1회. 2분 내 'aha'(관심자산 현재 국면+1줄 이유) | 철학3·4·7 | 활성화 24h 내·Day7 리텐션 +50%. <https://www.revenuecat.com/blog/growth/activation-metrics/> |

### 5-2. 배제한 다크패턴(이유)
- **가격 변동 푸시('OO 3% 급등!')** — Vanguard가 malicious로 직접 분류. 트레이딩 도파민의 핵심. 절대 금지.
- **게임화 리더보드·'인기 거래' 강조·뱃지 경쟁** — 위험행동·FOMO 유발(철학1·6 위반).
- **카운트다운·'지금 사세요/매도하세요' CTA** — 철학1·6 정면 위반.
- **스트릭 confirmshaming·복구 결제벽** — dark pattern, EU DSA·캘리포니아법 규제 리스크. 출처: <https://www.congress.gov/crs-product/IF12246>
- **근본원인 없는 '당신과 비슷한 사람들' 사회비교** — backfire·군중추종 위험.
- **거래량·DAU를 북극성 지표로** — 단기 인게이지먼트는 투자자 결과와 역상관(Barber & Odean 2000).

## 6. 인터랙션 플랜

> 2026 표준: **절제된 모션 + 성능 우선 + 접근성 기본 내장**. "거의 안 보이는 모션" — 모든 모션은 상태 변화/정보 위계를 *설명*할 때만 존재. 출처: <https://www.brad-holmes.co.uk/web-performance-ux/why-most-scroll-animations-miss-what-apple-gets-right/>

### 6-1. 라이브러리·셋업
| 용도 | 라이브러리 | 셋업 |
|------|-----------|------|
| 일반 모션 | **Motion(구 Framer Motion)** | `LazyMotion features={domAnimation} strict` + `m.*`만 사용(4.6KB, 기본 import는 34KB). 클라이언트 루트에 배치. <https://motion.dev/docs/react-reduce-bundle-size> |
| 전역 reduced-motion | **MotionConfig** | `reducedMotion="user"` — transform/layout off, opacity/색 유지. <https://motion.dev/docs/react-accessibility> |
| 숫자 트랜지션 | **NumberFlow** | 의존성 0, Intl ₩/% 포맷, reduced-motion 자동 존중. **결론 수치(비중%·도넛 합계)에만**, raw 시장 수치(원달러/VIX/DXY) 금지. <https://number-flow.barvian.me/> |
| 차트 | **Recharts(기설치)** | 도넛 최초 1회 reveal, 30분 갱신 시 `isAnimationActive={false}`. `role="img"`+`aria-label`(코드에 이미 있음). <https://blog.logrocket.com/best-react-chart-libraries-2026/> |
| 스크롤 리빌(랜딩) | **CSS scroll-driven**(`animation-timeline: view()`) | JS 0, 컴포지터에서 60fps. `@supports`+IntersectionObserver 폴백. <https://medium.com/@mihirzalavadiya1/stop-using-javascript-for-scroll-animations-css-quietly-killed-intersectionobserver-in-2026-e80c9a3937af> |

### 6-2. 구체 마이크로인터랙션
| 요소 | 인터랙션 | 타이밍 |
|------|----------|--------|
| 카드 호버/색 전환 | 미세 표면 명도 상승(surface-1→surface-2) | 150ms ease-out(`--dur-micro`) |
| '근거 펼쳐보기' 아코디언 | `grid-template-rows` 또는 Motion `layout` | 250ms ease-out(`--dur-base`) |
| 국면 카드 진입(대시보드) | `whileInView` + `viewport={{ once:true, amount:0.3 }}` fade/slide | 250~300ms |
| 비중%·도넛 합계 | NumberFlow 1회 차분히(깜빡임 금지) | — |
| 데이터 로딩 | **skeleton**(실제 레이아웃 모방, 슬레이트 톤, 느린 좌→우 shimmer). 인증/결제만 인라인 스피너 | — |
| 페이지 전환 | View Transitions API 또는 `template.tsx` 크로스페이드(슬라이드/줌/바운스 금지) | 200~250ms |

타이밍 표준: 마이크로 100~200ms, 일반 200~300ms ease-out. 80ms 미만 안 보임, 500ms 초과 굼뜸. `globals.css`에 `--dur-micro:150ms / --dur-base:250ms` 토큰화로 일관성. 출처: <https://www.appypie.com/blog/mobile-app-animation-guide>

### 6-3. 성능·reduced-motion
- CSS는 `@media (prefers-reduced-motion: no-preference)` **옵트인**(reduce 기본 → no-preference에서만 모션 추가). 약 35% 사용자/전정기관 장애 고려. <https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/At-rules/@media/prefers-reduced-motion>
- `transform`/`opacity`만 애니메이션(컴포지터 친화). `width/height/box-shadow`는 jank → grid-rows/Motion layout으로.
- 모션은 깜빡이며 멈추지 말고 안전하게 정지.

### 6-4. 배제(anti-Robinhood)
가격 숫자 깜빡임/플립·연속 펄스·폭주 티커·큰 패럴랙스·전 섹션 핀 고정 = 트레이딩 도파민. 모션은 '흥분'이 아니라 '변화 이해'에만. 출처: <https://medium.com/design-bootcamp/ux-tricks-from-robinhood-app-c485d6fba7a8>

## 7. 우선 구현 슬라이스 (로드맵)

> 원칙: **다크모드 먼저 → 그다음 랜딩 → 그다음 대시보드 풍부화**. 다크모드는 다른 모든 시각·인터랙션 작업의 선행 조건.

| 슬라이스 | 작업 | 산출물 | 의존 |
|----------|------|--------|------|
| **S1 다크모드 토큰(선행)** | `.dark` 블록 + `:root` 누락 토큰 + `darkMode:"class"` + tailwind 미등록 토큰 등록 + `REGIME_STYLES` CSS변수 전환 + no-flash 스크립트 + 토글(헤더/`/settings`) + 대비 재계산 | 다크 기본 동작, 컴포넌트 수정 최소 | 없음 |
| **S2 인터랙션 토대** | LazyMotion+`m`+strict + MotionConfig `reducedMotion="user"` + 모션 타이밍 토큰 + NumberFlow 도입(결론 수치) + Recharts `isAnimationActive` 분기 + skeleton | 절제된 모션 기반 | S1 |
| **S3 신선도 칩** | `cache_status`→상시 Live/Stale 칩(헤더 우상단) + 부드러운 transition + 다음 갱신 진행 | 정적 느낌 제거 | S1 |
| **S4 랜딩 신설** | `/` redirect→공개 랜딩(히어로 X-ray·국면 미리보기·가격 투명·CTA) + 미들웨어/SEO 재설계 | 첫인상·전환 진입점 | S1·S3 |
| **S5 대시보드 풍부화** | 국면 스펙트럼 + 한국 매크로 보드(4카드) + 점진적 공개 섹션 스택 + 빈 상태 + 넛지 카드 + 데스크톱 2열 | 허전함 해소 | S1·S2 |
| **S6 리텐션** | 온보딩 3스텝 + 이메일 다이제스트(결론무료/근거유료) + 국면 전환 nudge | 재방문 동력 | S4·S5 |
| **S7(후순위) 기업분석 요약 도형** | Snowflake형 RadarChart('근거 균형도' 프레이밍, 도형=Free·차원 raw=Pro) | 차별화 | MVP 5단계 |

각 슬라이스는 docs/01·02의 게이팅 매트릭스·Top-Down 순서·표현 금지어 사전을 그대로 준수한다.
