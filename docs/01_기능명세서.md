# THESIS 기능 명세서 (Feature Specification)

> 본 문서는 검증된 코드(`backend/app/*`, `frontend/lib|app|components/*`)와 기획 맥락을 교차검증해 작성했다.
> 추측으로 없는 기능을 만들지 않으며, 모든 tier 게이팅은 실제 코드 강제 지점을 근거로 한다.

---

## 1. 제품 개요 / 목표 / 타겟

### 1-1. 한 줄 정의
매크로 국면 → 자산배분 → 기업분석을 하나의 Top-Down 흐름으로 제공하는 **"투자 판단 근거"** SaaS. 한국 시장 특화.

### 1-2. 정체성
"지금 사세요"가 아니라 **"이런 근거들이 있습니다"**. 정답이 아닌 확률적 우위를 정직하게 제공하는 포지셔닝. 시장 예측 주장은 거짓말이라는 전제 위에서, 장기 신뢰를 차별점으로 삼는다.

### 1-3. 목표
| 구분 | 목표 |
|------|------|
| 제품 | 매크로 국면 결론을 입문자가 즉시 이해 → 그 위에서 자산배분 → (이후) 종목까지 하나의 흐름으로 연결 |
| 차별점 | (1) 매크로↔기업분석을 하나의 흐름으로 통합 (2) 원달러·외국인수급을 1순위로 둔 한국 특화 (3) DXY 수치 없이 결론을 파악하는 입문자 UX |
| 수익 | Free 유입 → Pro(₩9,900) 전환. 초기 유료 50명 = 월 ₩495,000. Pro 8명 ≈ 고정비($50~75/월) 손익분기 |

### 1-4. 타겟
| 타겟 | 플랜 | 니즈 |
|------|------|------|
| 입문자 | Free | DXY/표준편차 같은 raw 수치 없이 "지금 어떤 국면인지" 결론을 즉시 |
| 중급 개인투자자 | Pro | "왜 그런지" 지표 상세 근거 + 기업분석 + 포트폴리오 추적 |
| 퀀트/적극 투자자 | Quant | Pro + API + 커스텀 스크리너 + 백테스트 (미정의 — 후속 확장) |

### 1-5. 8개 설계철학 (모든 기능의 절대 기준)
| # | 철학 | 코드 강제 지점 |
|---|------|----------------|
| 1 | "지금 사세요" 금지 — 항상 "근거가 있습니다" 톤 | `Disclaimer.is_advice = Literal[False]`, '매도' 동사 부재(`models.py`) |
| 2 | Top-Down 순서 고정 (매크로→자산배분→기업분석) | `Layer` enum L1<L2<L3<L4 정렬, snapshot 레이어 오름차순 |
| 3 | 결론 먼저, raw 수치는 펼쳐보기 (입문자에 DXY 직접 노출 금지) | `RegimeConclusion`(수치 없음) vs `RegimeEvidence`(Pro) |
| 4 | 감정 언어 — 표준편차 대신 "-20% 빠지면 잠 못 잔다" | `config.risk_label_text` 5단계 감정 문구 |
| 5 | 실시간 데이터가 판단 기반 (어제 데이터로 오늘 판단 금지) | yfinance + TTL 캐시(30분), `cache_status` stale 추적 |
| 6 | 과신 방지 — 경고 임계값 도달해도 '매도' 금지 | `RegimeLabel`에 매도 라벨 부재, coverage<0.5 강제 중립, disclaimer 3중 |
| 7 | 결론 무료 / 근거 유료 | `evidence` Optional + `EvidenceLocked`, `json_schema_extra={'tier':...}` |
| 8 | 한국 시장 특화 (글로벌 툴 번역본 아님) | `config` L2(한국 매크로) layer_weight=1.6 (최고) |

## 2. tier 게이팅 요약 (전 카테고리 공통 기준)

> **원칙: 결론 = Free, 근거 = Pro, 자동화·확장 = Quant.**
> tier는 **Supabase `profiles.tier`에서만 서버 도출**(`auth.fetch_tier`, service_role, RLS 우회). 클라이언트 위조 불가. 익명/실패 시 **FREE fail-closed**.

| 데이터/기능 계층 | Free (₩0) | Pro (₩9,900/월) | Quant (₩29,900/월) | 코드 강제 지점 |
|---|---|---|---|---|
| 국면 **결론**(label/headline/방향) | ✅ | ✅ | ✅ | `RegimeConclusion` |
| 국면 **근거**(score/임계값비교/기여도) | 🔒 EvidenceLocked | ✅ `RegimeEvidence` | ✅ | `RegimeResult.evidence` Optional |
| **모든 16티커 raw 수치(latest/prev_close/SMA)** | 🔒 마스킹(None) | ✅ | ✅ | `_mask_snapshot_for_free`(routes.py:140) |
| 16티커 change_pct·status·추세 라벨(요약) | ✅ | ✅ | ✅ | `_free` 필드는 마스킹 제외 |
| 단순 자산배분 결론 | ✅ | ✅ | ✅ | `AllocationConclusion` |
| 자산배분 근거(국면이유/한국신호/제약) | 🔒 EvidenceLocked | ✅ `AllocationEvidence` | ✅ | `AllocationResult.evidence` |
| 외국인수급 series(시계열) | 🔒 None | ✅ | ✅ | `_mask_snapshot_for_free` |
| 기업분석 전체 / 포트폴리오 추적 | ❌ | ✅ | ✅ | (미구현) |
| 커스텀 스크리너 / API / 백테스트 | ❌ | ❌ | ✅ | (미구현) |

### ⚠️ 게이팅 정정 (코드 직접 검증 결과 — 중요)
- **`free_visible` 플래그는 게이팅에서 무시되는 dead flag다.** `tickers.py`는 VIX·원달러·코스피에 `free_visible=True`를 달고 주석에 "무료 플랜에서 원시 수치까지 노출"이라 적었으나, **실제 `_mask_snapshot_for_free`(routes.py:140-154)에는 `free_visible` 분기가 전혀 없다.** 전 metric의 `latest`/`prev_close`/`error`와 `trend.sma5/sma20`를 일괄 `None`으로 마스킹한다(코드 직접 확인).
- 따라서 **"기본 지표 3개 raw 무료"는 사실이 아니다.** Free는 VIX/원달러/코스피를 포함해 16티커 전부 **change_pct·status·추세 라벨(요약)만** 보고, **raw latest 수치는 전부 Pro 전용**이다.
- **결정 채택: (B) 기획을 코드에 맞춤** — 본 문서는 철학3("입문자에게 DXY 등 raw 수치 직접 노출 금지")을 우선해 "기본 지표 3개는 요약만 무료, raw는 Pro"로 명세한다. `free_visible` 플래그는 정리(제거) 또는 명시적 의미 재정의 대상으로 표기(→ open_questions).

### ⚠️ Tier enum 한계
- 현재 `Tier` enum(`models.py`)과 프론트 `types.ts`는 **`free`/`pro`만** 정의. `profiles` CHECK(0001_init.sql)도 `in ('free','pro')`. `supabase_admin.set_user_tier`도 free/pro만 허용.
- **Quant 전 기능은 tier 값 자체가 미정의** → ① DB CHECK ② Tier enum(BE `models.py` + FE `types.ts`) ③ `set_user_tier` 허용목록 ④ Stripe price **4곳** 확장이 선행 작업.

### 4-1. 알려진 결함 (명세 반영)
- **`free_visible` dead flag**: 마스킹 로직이 플래그를 무시 → "기본 지표 3개 raw 무료"는 사실 아님. 플래그 정리 또는 분기 추가 결정 필요.
- **`getIndicators` 토큰 미전달**: 현재 시그니처로는 Pro도 raw 못 봄 → `/indicators` 구현 시 accessToken 인자 추가 필수.
- **`health.cache_warm` 키 불일치**: `'snapshot'` vs 실제 `'snapshot:all'` → 항상 False(경미).
- **regime 미채점 7티커**: 금/WTI/^IRX/주식4종 indicator_rule 부재.
- **`Tier` enum**: free/pro만 → Quant는 DB CHECK/enum(BE+FE)/set_user_tier 허용목록/Stripe price **4곳** 확장 선행.
- **OverconfidenceBanner 위치**: 코드상 dashboard 최하단 → 철학6 효과 약화, 상단 이동 필요.
