# 09. THESIS 대규모 리팩토링 — 마스터 플랜

> 생성: ultracode 다관점 워크플로(현 구조 파악 → IA·인게이지먼트·비주얼·전환 4관점 병렬 설계 → 종합+완결성 비평, 8 에이전트).
> 근거: [docs/08 딥리서치](08_사이트_리팩토링_리서치.md)(검증된 패턴). 2차 딥리서치(전환 메커니즘·2025–26 비주얼/모션·한국 비교사례)는 §9 표의 단계를 후속 보강.
> 진단: 네 관점이 같은 4개 문제에 수렴 — (a) 평평한 정보 격자 (b) 재방문 동기 0 (c) 잠금/배너/카드 마크업 산재 (d) "compared to what?" 부재. → **공유 프리미티브 한 벌**이 넷을 동시 해결.

## 1. 비전 / 원칙 (왜 이 방향인가)

THESIS의 본질 문제는 "정보 부족"이 아니라 **"정보가 행동으로 직조되지 않음"**이다. 탭들이 stateless 단발 조회라 재방문해도 첫 화면과 같고, 결론이 화면마다 형태가 다르며, Pro 가치가 텍스트 한 줄로만 암시된다.

검증된 5대 방향:
1. **결론 먼저, 근거는 "보이는" 펼침**(NN/G Progressive Disclosure) — THESIS "결론먼저" 철학이 정답임이 리서치로 확증. 단 ≤2단계·discoverability 필수, **위험경고·신선도는 절대 숨기지 않음**.
2. **단일 평결 + 차원별 상태**(Seeking Alpha) — 평평한 16지표 격자를 상태 칩 격자로.
3. **모든 경험을 물 흐르듯 연결**(Toss증권, 동일 시장·입문자) — Top-Down(매크로→배분→기업)을 화면 안 인과 흐름으로.
4. **인게이지먼트 = 정보가 아니라 행동 비계** — 좋은 투자자=반복 루틴. 관심종목·임계값 구독·저장 규칙·자동 최근본종목으로 day-7 재방문 트리거.
5. **"Compared to what?"** — 모든 숫자에 비교 랜드마크(임계값)+절제된 delta. 입문자에겐 raw 대신 "어느 구간에 서 있나".

불변 제약(모든 Phase 게이트): Top-Down 고정 / "사세요" 금지·과신방지 상시 / 결론무료·근거유료 / 한국 1순위 / raw 비노출 / any·하드코딩 금지·접근성 / 스택 고정.
기각(적용 금지): Robinhood 미니멀리즘, 게임화(컨페티·점수·연속기록), 핀테크 리텐션 수치, F/Z패턴 배치근거, PD 4기법 분류.

## 2. 공유 디자인 프리미티브 (빌딩블록)

**2-A 레이아웃**: `ui/Panel`(카드 하드코딩 흡수, tone) · `ui/NoticeBanner`(Disclaimer·Overconfidence·Stale·SessionWatch 공통 골격, 기존 4배너는 thin wrapper) · `ui/ModalShell`/`Drawer`(backdrop+focus-trap+Esc+aria) · `ui/EmptyState`(teaching/soon — "준비중" 아니라 "이렇게 시작").
**2-B 상태/등급**: `ui/StatusPill`(ThresholdHit calm/neutral/warn/danger → 색+라벨+점, **수치 비노출·색 단독 금지**) · `ui/Chip` · `ui/ProBadge` · `ui/FreshnessChip`(승격) · `ui/LockedValue`(raw 자리 `•••↑`, 가짜 숫자 금지).
**2-C 데이터비주얼(평평함 해소 핵심)**: `lib/viz/scale.ts`(색맵·delta·status→색 단일출처, regimeTokens 흡수) · `viz/ThresholdGauge`(**"compared to what" 주력**, 밴드+마커, 숫자없이 위치만 Free-safe) · `viz/Sparkline`(Free=도식/Pro=실series) · `viz/BulletRow`(목표·임계 대비) · `viz/DeltaText`(Free 정성/Pro 정량).
**2-D conclusion/glance**: `conclusion/ConclusionCard`(평결 골격, RegimeSignalCard·Allocation·기업이 thin wrapper) · `ConfidenceMeter` · `glance/PageConclusion`(탭 상단 슬림 헤더) · `glance/GlanceHub`("오늘" 허브) · `glance/KoreaTriadStrip`(원달러·코스피·VIX 3칩) · `glance/NextStep`(Top-Down 흐름 네비).
**2-E metric 통합**: `ui/MetricCard` ← MacroMiniCard+TickerCard+ForeignFlowCard **3종 통합**(StatusPill+Sparkline+ThresholdGauge+LockedValue+AlertRuleControl 슬롯).
**2-F 페이월/업셀**: `ui/BlurTeaser`(실 콘텐츠 흐림, raw는 뒤에도 없음) · `paywall/LockedCard`(EvidenceLocked+LockedPreview 통합) · `billing/UpgradeModal`(인컨텍스트, source 추적) · `billing/PlanComparison`(단일출처 lib/plans) · `billing/UpsellInline` · `personalization/{WatchlistButton,AlertRuleControl,SavedRuleCard,DigestList}`.
**2-G 단일출처 모듈**: `lib/plans.ts` · `lib/upsell.ts` · `lib/analytics.ts`(no-op-safe) · `lib/viz/scale.ts` · `lib/personalization/*`(Supabase 직접, lab_canvases 패턴). 전부 `any` 금지.

> 충돌 조정: `ConclusionCard`=내용 골격 / `PageConclusion`=그걸 쓰는 탭 상단 슬림 변형(계층 관계, 중복 아님).

## 4. 단계별 플랜 (Phase 0..7) — 각 단계 = 독립 배포 단위, 빌드 통과 유지

- **Phase 0 — 기반(프리미티브+백엔드 토대) · 무행동변경**: 토큰(`--status-*`), `lib/viz/scale`·`lib/plans`·`lib/upsell`·`lib/analytics`, 2-A/2-B 전부 + 기존 하드코딩 치환, **BE1 threshold_status**(FE 무변경 선행) + BE3 계측 스캐폴드(no-op). 외형 동일·회귀 최저.
- **Phase 1 — MetricCard 통합 + 데이터비주얼 · 평평함 해소 본체**: MetricCard 3종 통합, viz/{ThresholdGauge,Sparkline,BulletRow,DeltaText}, Disclosure(`<details>`), BE1 소비. /indicators·대시보드 한국지표 시각 격상.
- **Phase 2 — conclusion 통합 + 흐름 직조**: ConclusionCard/ConfidenceMeter(RegimeSignalCard·Hero·Allocation), PageConclusion/NextStep를 탭 상하단, AllocationPanel BulletRow(현금 vs 20%·단일 vs 15%).
- **Phase 3 — 페이월/업셀 통일**: BlurTeaser·LockedCard·UpgradeModal·PlanComparison·UpsellInline, EvidenceLocked 리라이트, BE5 소비. **`/lab` 모달은 단독 분리·충분검증**(BlockCanvas React Flow 회귀 위험).
- **Phase 4 — GlanceHub(dashboard "오늘" 재편) · 가장 가시적**: GlanceHub·KoreaTriadStrip, 2초 글랜스(국면+한국3+배분1줄+넛지), ②③④ detail 강등, Sidebar 라벨 "오늘".
- **Phase 5 — 개인화 상태층**: BE2 + lib/personalization, WatchlistButton·recently_viewed, /portfolio→"내 관심"(EmptyState teaching), SavedRuleCard, WelcomeLetter 온보딩, /screener 교육형.
- **Phase 6 — 알림·다이제스트 + 활성화 배선**: BE4 + AlertRuleControl(Pro, Free 1개 teaser) + DigestList + source별 upsell 계측.
- **Phase 7 — 마감**: 모바일 단일컬럼(명시 order), 라우트별 loading.tsx 스켈레톤, 모션 헌장(카운트업 제거·opacity 전환·reduced-motion 전수), is_activated 3-test.

> 병렬 가능: BE1·BE3(P0), 페이월(P3)은 개인화(P5)와 독립. `/pricing`은 P3 직후 단독 배포.

## 6. 상시 가드 점검표 (머지 게이트)
**Free/Pro**: StatusPill·threshold_status·게이지·Sparkline(도식)·locked_summary·온보딩·관심종목/저장 1개=Free; raw level·근거상세·기여도·알림규칙·실시간·무제한=Pro. tier는 **서버 응답에서만**(`regime?.tier ?? allocation?.tier ?? "free"`). 개수 상한은 RLS/백엔드 강제. 활성화 위해 **alert_rule Free 1개 허용**(teaser).
**접근성**: 색 단독 금지(텍스트+아이콘+sr-only), hover 전용 금지(tap/focus, aria), ModalShell/Disclosure aria+focus-trap+Esc, 게이지 role=img+aria-label, BlurTeaser aria-hidden+SR "Pro 잠금:{label}", reduced-motion, 대비 4.5:1.
**과신방지**: "이런 근거가 있습니다" 톤, OverconfidenceBanner glance 상시(업셀 모달 안에서도), danger→"주의/경계"(팔아라 연상 차단), 카운트업 제거·delta/색 절제, 게임화 금지.
**raw 숨김**: DXY/VIX raw는 Pro 마스킹, 게이지 마커 **밴드 중앙 스냅**(raw 역산 차단), locked_summary/StatusPill/DeltaText(Free)=라벨만, BlurTeaser 뒤 실 raw 없음(`•••`만), **위험경고·신선도는 숨기지 않음**.

## 7. 활성화 / 리텐션 계측
**활성화 정의(확정)**: "가입 7일 내, **자산배분 1회 실행 AND 관심종목 ≥1**". (자산배분=제품 이해 순간[전환 선행], 관심종목=재방문 시 볼 것[리텐션 선행]. 알림은 Pro 게이팅이라 필수조건 제외, 보조 트리거.)
**측정**: `activation_allocation_run`+`watchlist_add` → usage_events → `is_activated` 뷰(7일 윈도). **3-test(단정 금지)**: ①활성화 코호트 day-7 재방문 유의↑ ②활성화 중앙시간<7일 ③활성화율≥신규 7%(상위25%). 보조: paywall_hit/upsell_view/click(source별). PII 금지·fail-open.

## 8. 피할 것 (기각·금지)
Robinhood 미니멀리즘 / 게임화 일절(컨페티·점수·연속기록·뱃지·보상연출) / 핀테크 93–96%·2주98% 통계 / F·Z패턴·Density Disjoint·PD 4기법 / "사세요·Pro면 더 잘 산다" 카피 / 가짜 데이터(BlurTeaser·placeholder·Sparkline에 거짓 수치) / 모든 카드 업셀 부착(막힌 지점에만).

**게임화 do-not-build 확정 목록(2차 딥리서치 [docs/10](10_딥리서치2_전환_리텐션_안티패턴.md) — 규제·학술 하드근거):** "biggest mover" 변동성 푸시 · 랜덤 서프라이즈 종목 · 리더보드 · 스트릭/콘테스트(경품) · 포인트/뱃지 · 컨페티 — 영구 금지. 근거: FCA 9천명 실험(푸시 → 거래 +11%·위험거래 +8%, **입문자 최대 피해**), Yale LJ 금지목록, Barber&Odean(과매매 11.4% vs 시장 17.9% — 과신이 손실 원인). 알림은 **임계값 돌파 통지에 한정**(무버/실시간 푸시 금지). 전환 **수치 벤치마크는 전부 미검증** → 수치로 정당화 금지.

## 완결성 점검 (자기 비평 → 보완)
1. **기업분석(CompanyFundamentals) 시각화·Free/Pro 경계가 얇음** — DART/yfinance 재무에 tier 메타 없어 무단 raw 노출 위험. → Phase 2에 "기업 ConclusionCard(평결+차원 등급)", Phase 3에 "기업 재무 블록 Pro 게이팅(프론트 명시)" 추가.
2. **테스트/회귀 전략 부재** — "빌드 통과"만으론 프리미티브 치환·마스킹 회귀 못 잡음. → Phase 0에 (a) 마스킹 단위테스트 (b) 배너 wrapper 톤 매핑 검증 (c) BlurTeaser DOM raw 부재 검증 게이트 추가.
