# 07. 블록 등급 & 과금 모델 — "결핍 사다리"

> **상태: 확정 (2026-06-18).** 앞으로 모든 블록 추가는 5장 rubric으로 등급을 매긴다.

> 목적: THESIS의 **핵심 결제 모델**을 확정한다. (a) 왜 "결핍 사다리"가 옳은지(이론·업계 근거), (b) 새 블록을 추가할 때마다 재사용할 **등급 채점 기준표(rubric)**, (c) 현재·예정 블록에 적용한 Free/Pro/Quant 배치.
> 4갈래 리서치(전환심리 / 투자툴 벤치마킹 / SaaS 버저닝 / 노드툴 게이팅) 종합. 모든 출처는 실제 URL로 검증.
> 사용자 핵심 의도: *"입문자는 쉽게 또 깊이있게(퀀트도). 블록을 용도별 차등 등급. 각 단계에 의도적 결핍 — 초보가 쓰다 더 전문 지표를 보려 하면 돈으로 막혀 결제를 유도."*

---

## 0. 한 줄 결론

**무료 블록 = 유료 블록의 "의도적으로 축소된 버전"**이다. 결핍은 사고가 아니라 설계다. 무료로 *결론*까지 도달하게 한 뒤, 그 결론 **바로 옆**에 *근거* 블록을 흐릿하게 보여줘 호기심 격차(curiosity gap)를 최대화하고, 사용자가 "더 깊이 보고 싶다"고 느끼는 **정확히 그 지점**에 paywall을 둔다. 이건 THESIS의 기존 철학("결론 무료, 근거 유료")이 경제학 이론(versioning / damaged goods)과 업계 규범(Morningstar·Simply Wall St) 양쪽으로 검증된다는 뜻이다.

## 1. 핵심 원리 — 결핍 사다리의 이론적 근거

사용자가 직관으로 도달한 "의도적 결핍" 아이디어는 정확히 두 경제학 이론이다:

### (1) Damaged Goods (의도적 열화로 가격차별)
- Deneckere & McAfee, *"Damaged Goods"* (1996) — 기업이 **일부러 열화된 버전**을 만들어 지불의향(WTP)으로 소비자를 분리하는 것이 **파레토 개선**일 수 있다(원래 서비스 못 받던 사람도 싸게/무료로 받게 됨). → 무료 블록으로 입문자를 들이는 것은 윤리적으로도 정당.
  - 랜딩(오픈): https://ideas.repec.org/a/bla/jemstr/v5y1996i2p149-174.html
- McAfee, *"Pricing Damaged Goods"* (2007, 오픈 전문): https://www.econstor.eu/bitstream/10419/17999/1/economics_2007-1.pdf

### (2) Versioning (정보재는 버전 메뉴로 자기선택 유도)
- Shapiro & Varian, *Information Rules* + HBR *"Versioning: The Smart Way to Sell Information"*(1998). 핵심: **고품질 버전을 먼저 만들고, 의도적으로 줄인 하위 버전**을 만들어 소비자가 스스로 그룹을 나눠 들어가게 한다.
  - Varian, *"Versioning Information Goods"*(오픈 PDF): https://people.ischool.berkeley.edu/~hal/Papers/version.pdf
  - 버전 분리 축(직접 적용 가능): **시의성(실시간 vs 지연), 기능/역량, 포괄성, 편의성, 지원, 속도.** Varian은 금융정보재의 대표 축으로 **실시간 vs 지연 시세**를 명시 — THESIS 시장데이터에 그대로 적용.
  - HBR 버전: https://faculty.haas.berkeley.edu/shapiro/version.pdf

> **결론:** "결핍 사다리"는 검증된 전략이고, 입문자를 무료로 들이는 것은 welfare-improving. 단 결핍은 *정직*해야 한다(아래 8장).

## 2. Paywall 배치 원칙 — "aha 이후"

- **활성화(aha) 이후, 과업 완료 이전에 막는다.** aha 모먼트를 이미 느낀 유저는 약 **5배** 더 결제한다. → 무료로 *결론*까지 도달하게 하고, 그 다음 *근거/깊이*에서 막는다.
  - RevenueCat 2026 벤치마크(11.5만 앱): hard paywall D35 전환 10.7% vs freemium 2.1%(~5x). 단 12개월 결제유지율은 27% vs 28%로 비슷 — 차이는 *속도*지 *점착성*이 아님. https://www.revenuecat.com/blog/growth/subscription-app-trends-benchmarks-2026/
- **THESIS 적용: Free→Pro 벽 = "근거/깊이 경계".** 무료 블록은 *무엇(국면 신호·단순 자산배분)*을 말하고, 바로 옆 Pro 블록이 *왜(DXY·VIX·원달러 디테일, 자산별 근거)*를 쥔다. ← "결론 무료, 근거 유료"와 정확히 일치.
- **Pro→Quant 벽 = "규모·자동화·커스터마이즈 경계"**(깊이가 아님 — Pro에서 이미 깊이는 충분). 커스텀 스크리너·백테스트·API·대량 캔버스. TradingView 사다리와 동형(전문 깊이는 일찍, *전문가급 규모*는 최상위).
- **Free 관대함의 균형:** 무료는 *완결된 저해상도 분석*을 줘야 한다(실제 가치 → 유지·입소문). "모두가 필요한 기능이면 무료, 파워유저만 필요하면 업그레이드 레버." (Stripe) https://stripe.com/resources/more/freemium-pricing-explained
  - freemium 무료→유료 전환 벤치마크: self-serve 3~5% 양호 / 6~8% 우수. https://www.lennysnewsletter.com/p/what-is-a-good-free-to-paid-conversion

## 3. Teaser / 미리보기 메커니즘 — "보여주되 잠근다"

결핍이 동기가 되려면 잠긴 블록이 **보여야** 한다. 작동 심리:

- **호기심 격차(Loewenstein 1994):** 호기심은 "지식의 격차에 대한 인지적 박탈감". 작은 정보 한 조각이 격차를 메우려는 충동을 급격히 키운다(부분 지식에서 호기심 최대). → 잠긴 블록의 *형태/제목*만 보여주고 값은 가린다. https://www.cmu.edu/dietrich/sds/docs/loewenstein/PsychofCuriosity.pdf
- **Zeigarnik 효과:** 미완의 과업은 긴장을 만들어 사람을 다시 끌어당김. "잠겼지만 보이는 다음 단계"가 이 열린 고리를 만든다. https://www.ux-bulletin.com/zeigarnik-effect-ux/
- **손실회피(Kahneman·Tversky):** 손실은 이득의 ~2배로 느껴짐. 프리미엄 역량을 한 번 맛보면 빼앗기는 게 손실로 느껴져 WTP↑. https://thedecisionlab.com/biases/loss-aversion
- **보유효과:** 잠깐이라도 "내 것"이 된 기능은 ~2배 가치. 1회 미리보기 → 부드러운 재잠금. https://ixdf.org/literature/article/endowment-effect-the-economics-of-design

### 업계의 teaser 구현 패턴 (노드툴에서 직수입)
- **잠긴 블록을 팔레트에 배지로 노출**(숨기지 말 것). 항상 광고로 작동(Notion이 AI·자동화 행 노출, Figma가 Dev Mode 노출).
- **읽기 허용 / 실행·저장 차단(read-allowed, run-gated).** 블록을 드래그·설정은 되나 *출력*이 블러/샘플 + 인라인 "Pro로 펼치기" CTA(Bubble=빌드 무료·게시 유료, Airtable=보기 무료·API/규모 유료).
- **업그레이드 프롬프트는 "의도 시점"에**(과업 도중 금지). Figma 정석: 에디터 추가/고급 권한 시도 시 = 이미 만들 가치가 생긴 뒤라 자연스러움. https://www.appcues.com/blog/best-freemium-upgrade-prompts
- **soft limit(멈춤, 에러 아님).** 한도 초과 시 추가 노드가 "Pro"로 표시될 뿐 에러 안 남(Airtable 자동화 정지 패턴).

## 4. 업계 벤치마크 요약

### 투자분석 툴 — 3대 패턴
1. **결론/신호는 Free, 근거 디테일은 유료** — Morningstar(별점 무료 / 공정가치·해자·풀리포트 유료), Simply Wall St(Snowflake 점수 노출 / 전체 DCF는 리포트 열람횟수로 메터링). **THESIS 철학과 정확히 일치.**
   - https://www.morningstar.com/mm/investor/stock-analysis · https://simplywall.st/plans
2. **첫 유료(Pro)의 단골 해금 4종**: 커스텀 스크리너 · 데이터 히스토리(→10년) · 실시간 데이터 · 익스포트.
   - Finviz(커스텀 필터=Elite) https://finviz.com/elite.ashx · Koyfin(히스토리 2→10년) https://www.koyfin.com/pricing/ · TradingView https://www.tradingview.com/pricing/
3. **최상위(Quant)의 단골 해금 4종**: 무제한 커스텀 수식/메트릭 · 백테스팅 · API · 최장 히스토리(20~30년)·대량 스크리너.
   - Stock Rover(커스텀 메트릭 0→30→무제한) https://www.stockrover.com/plans/ · GuruFocus(백테스트=상위 티어) · YCharts(API=Enterprise) https://get.ycharts.com/plans/
   - Stockopedia: 팩터 점수(StockRanks) 자체를 통째로 유료 게이팅(더 공격적 사례) https://www.stockopedia.com/plans/

### 노드/블록 툴 — 코드/수식 블록의 2진영
> 가장 설계 관련 발견.
- **A진영(코드/수식 = 무료, 획득 미끼):** n8n(Code 노드 전 티어 무료), Retool(JS/SQL 무료), Coda(수식 무료), Bubble(API Connector·JS 무료, 대신 *게시* 차단). → 청중이 *기술자*라 코드가 미끼, 차별점은 규모·거버넌스·배포.
  - https://docs.n8n.io/integrations/builtin/core-nodes/n8n-nodes-base.code/ · https://docs.retool.com/queries/guides/transformers
- **B진영(코드/스크립트 = 유료, 전환 레버):** Airtable("Run a script"=Team+), Make(custom functions=Enterprise), Zapier(Code=유료 멀티스텝), Notion(Workers 커스텀코드=Business+). → 청중이 *비기술자*라 코드 블록이 "무료를 졸업했다"는 신호.
  - https://support.airtable.com/docs/getting-started-with-airtable-automations · https://help.make.com/custom-functions · https://www.notion.com/help/run-custom-code-with-workers

> **THESIS는 입문자 우선 = B진영.** 따라서 **커스텀 수식 블록은 자연스러운 유료 파워 블록**(Quant).

### 3가지 게이트 유형 (전 툴 공통)
1. **규모 천장(usage gating)** — 레코드/실행/워크로드/히스토리 일수/파일크기. 가장 보편적.
2. **게시·협업(outcome gating)** — 만들 가치가 생긴 *뒤*에 필요 → 전환율 높음(Bubble 게시, Figma 에디터 추가).
3. **거버넌스/신원(feature gating, 최상위)** — SSO·SCIM·감사로그·RBAC. 빌더가 아니라 구매자에게 판매. (THESIS는 당장 무관)

## 5. ★ 블록 등급 채점 기준표 (rubric) — 재사용

새 블록을 추가할 때마다 이 표로 채점한다. **7개 항목 × 0~2점 (최대 14점).**

| # | 항목 | 0점 | 1점 | 2점 |
|---|------|-----|-----|-----|
| **D1** | **깊이/복잡도**(파워유저 가치) | 입문자용, 단순 조회 | 약간의 해석 필요 | 깊은/고급, 파워유저만 가치 추출 |
| **D2** | **전문/상업용** | 개인·교육용 | 혼합 | 명확히 전문·상업 의사결정(포트폴리오 운용, 고객업무) |
| **D3** | **규모/사용량 민감도**(usage-fence 적합) | 가치 고정·1회성 | 약간 스케일 | 양(다종목·다백테스트·다포트폴리오)에 따라 가치 급증 |
| **D4** | **시의성/데이터 신선도**(Varian 축) | 정적/지연 OK | 주기 갱신 도움 | 실시간이 본질 |
| **D5** | **커스터마이즈/제어** | 없음 | 가벼운 프리셋 | 완전 커스텀(커스텀 스크리너·가중치·API/익스포트) |
| **D6** | **결제의향(WTP) 신호** | 결제자와 무관 | 약함 | 이 블록 헤비유저 = 결제하는 사람 |
| **D7** | **획득/습관/바이럴 가치**(역방향 — 높으면 무료 유지) | 가입·일상습관·공유 유발 | 중립 | 니치, 획득·습관 가치 없음 |

**추가 판정자(채점 외):**
- **과신 위험 플래그(저/중/고):** 자작 Score·백테스트·레버리지/사이징 도구 = *고*. → (a) 최소 Pro로 상향, (b) 출력 전 경고 컴포넌트 강제. (과금 로직과 윤리 가드를 같은 방향으로)
- **한계비용 tie-breaker:** 실시간 피드·무거운 백테스트처럼 호출당 데이터/연산비 높으면 점수와 무관하게 한 티어 상향(단가 보호).

### 매핑 규칙
1. **합계 계산(0~14)** 후 *프로필*도 확인 — 한 항목이 결정적이면 합계를 덮어쓴다:
   - **하드 Pro/Quant 트리거(합계 무시):** D4=2(실시간) **또는** D5=2(API/커스텀 익스포트) **또는** D6=2(강한 WTP) → **최소 Pro**.
   - **하드 Free 트리거:** D7=0(가입/습관/공유 유발) **그리고** D1≤1 **그리고** D2=0 → 합계가 중간이어도 **Free 유지**(획득/습관 코어는 막지 마라 — "권력을 막되 접근은 막지 마라").
2. **그 외 합계로 매핑:** **0~4 → Free** / **5~9 → Pro** / **10~14 → Quant**.
3. **Free 완결성 점검(과/소 게이팅 방지):** 배치 후, *무료 블록만으로 핵심 결론(국면 신호 verdict)에 paywall 없이 도달*하는지 확인. 안 되면 한 블록을 Free로 강등. 반대로 Free만으로 Pro가 할 걸 다 하면 한 사용량 축을 Pro로 승격.
4. **단일 점프 가드:** 무료 UI에서 처음 마주치는 곳보다 두 티어 위에 두지 말 것. Quant 블록이 Free 화면에 보이면 *미리보기*(아이콘/툴팁/샘플)로 노출하되 **숨은 paywall 금지**(신뢰 손실 1위 안티패턴).

### 채점 예시 (THESIS)
- **국면 신호 verdict 카드** → D1:0 D2:0 D3:0 D4:1 D5:0 D6:0 D7:0 = 합 1 + Free 트리거 → **Free** ("결론 무료").
- **지표 상세/근거(DXY·VIX 수치)** → D1:1 D2:1 D3:1 D4:1 D5:0 D6:2 → D6=2 오버라이드 → **Pro** ("근거 유료").
- **커스텀 스크리너+백테스트+API** → 합 13 + D5/D6 오버라이드 → **Quant**.

## 6. THESIS 블록 등급 적용표

| 블록 / 기능 | 등급 | 근거 |
|---|---|---|
| **기업 블록**(단일 기업 기본 데이터) | **Free** | 캔버스를 살아있게 만드는 핵심 aha 블록. 업계 전반 단일기업 기본지표 무료 노출 |
| **기본 메트릭**(ROE·PER·영업이익률 등 3개) | **Free** | 획득 미끼, "결론 무료" 철학, 업계 Free 표준 |
| **고급/밸류 메트릭**(PEG·DCF·공정가치) | **Pro** | 밸류 깊이는 보편적 첫 유료(Morningstar 공정가치, SWS DCF) |
| **비교 표**(다종목) | **Free(소수·기본컬럼) / Pro(무제한·깊은컬럼)** | 비교는 풀되 깊이/종목수로 게이팅(규모 천장) |
| **랭크/팩터 점수**(가중 국면·팩터 스코어) | **Pro** | 합성 *신호/결론*은 Free, *산출 디테일*은 Pro(Morningstar 별점 vs Stockopedia) |
| **스크리너/필터** | **Free(프리셋) / Pro(커스텀) / Quant(대량·고급)** | Finviz 정석: 프리셋 Free, 커스텀값 Elite |
| **가중 Score 블록**(사용자 가중) | **Pro(소량) → Quant(복합·저장형)** | YCharts Scoring=Pro+. 과신 위험 플래그 *고* → 경고 강제 |
| **커스텀 수식 블록** | **Quant**(Pro에 소량 허용) | B진영 논리. 무제한은 거의 항상 최상위(Stock Rover 무제한=Ultimate). 2단 게이트 권장 |
| **백테스팅** | **Quant** | 일관되게 최상위(GuruFocus·Finviz Elite). 과신 위험 플래그 *고* |
| **익스포트** | **Pro(CSV) / Quant(PDF·대량·자동)** | Notion 정석(CSV 하위 / 대량 PDF 최상위) |
| **API 접근** | **Quant**(Pro에 throttled teaser 옵션) | 예외 없이 최상위(YCharts Enterprise, Airtable 1,000콜 teaser) |
| **데이터 히스토리 깊이** | **Free 2~3년 / Pro 10년 / Quant 20년+** | 가장 보편적 게이트(Koyfin·TIKR·GuruFocus 단계화) |
| **실시간 데이터** | **Pro** | Free 지연, 실시간 첫 유료(Varian 시의성 축) |
| **공유 / 마켓플레이스** | **보기 Free / 공유 Pro / 마켓 게시 Quant** | outcome gating. 보기 무료로 바이럴, 게시는 유료 |

> **기존 기획 정합성:** CLAUDE.md의 (국면 신호 Free / 지표 상세 Pro / API·백테스트 Quant)는 Morningstar(신호 무료·근거 유료) + Stockopedia(팩터 유료)의 절충형으로 **업계 규범과 잘 맞음.** 유일한 조정 권고 = **커스텀 수식/가중 Score는 "Pro 소량 + Quant 무제한" 2단 게이트.**

## 7. 결핍 사다리 — 사용자 여정 (이게 결제를 만든다)

```
[입문자 · Free]
  기업 블록 + ROE·PER 보기 → 국면 신호 → "오, 지금 경계 국면이구나" (aha, 무료)
        │  옆에 흐릿한 근거 블록: "VIX 추세 · 원달러 10일 · 외국인 수급 🔒 Pro"
        ▼  "왜 경계인지 보고 싶은데?" ← 호기심 격차 최대 지점
[중급 · Pro]
  근거 디테일 + 다종목 비교 + 커스텀 스크리너 + 실시간
        │  "삼성 vs 하이닉스 vs 엔비디아를 내 기준으로 점수 매기려면?"
        ▼  커스텀 수식/대량 스크리너 시도 → 🔒 Quant
[전문가 · Quant]
  커스텀 수식 무제한 + 백테스트 + API + 마켓 게시
```
각 경계 = 사용자가 *스스로 더 깊이 가고 싶다*고 느끼는 순간. 막는 게 아니라 *맛보게 한 뒤* 막는다.

## 8. 철학 가드 — 과금 UX도 "지금 사세요" 금지

투자 UX의 "사세요 금지" 원칙이 **업셀 UX에도 그대로** 적용된다. 근거로 권하되 압박/거짓긴급 금지:

- **잠금 문구 = 역량, 명령 아님.** "이 근거를 펼쳐보려면 Pro" ✅ / "지금 업그레이드하세요!" + 카운트다운 ❌. ("이런 근거들이 있습니다" 톤과 동형)
- **1회 정직한 맛보기**(보유효과) 후 부드러운 재잠금 — bait-and-switch 금지.
- **dark pattern 금지:** 강제 연장(forced continuity), 가짜 희소성, 취소 경로 숨김 금지. A/B만 보면 기만으로 치우치니 정성 피드백 병행. https://alistapart.com/article/dark-patterns-deception-vs-honesty-in-ui-design/
- **잠긴 미리보기도 3중 disclaimer 유지** — teaser가 분석 출력을 보이므로 과신 방지 고지 상시.
- **과신 위험 블록(자작 Score·백테스트)은 결제 + 경고 둘 다.** 돈을 내도 경고는 안 사라진다(survivorship/overfitting 경고 강제).

## 9. 구현 체크리스트 (블록 시스템 단계와 연동)

- [ ] 블록 정의에 `tier: "free"|"pro"|"quant"` + `overconfidenceRisk?: "low"|"med"|"high"` 필드
- [ ] 잠긴 블록 팔레트에 배지 노출(숨기지 않음) — 등급별 색/아이콘
- [ ] 잠긴 블록 드래그·배치·설정 허용, **출력만 블러/샘플 + 인라인 CTA**(read-allowed, run-gated)
- [ ] 업그레이드 모달 = "실행/계산/N+1번째 기업 추가" 클릭 시(의도 시점, aha 이후) — 배선 도중 방해 금지
- [ ] soft limit(멈춤·"Pro" 표시, 에러 금지)
- [ ] 과신 위험 *고* 블록: 출력 전 경고 컴포넌트 강제 렌더
- [ ] Free 완결성 자동 점검: 무료 블록만으로 국면 신호 결론 도달 가능 여부
- [ ] tier는 서버 profiles에서만 도출(기존 패턴 유지), 클라이언트 신뢰 금지

> 새 블록 추가 시: **5장 rubric으로 채점 → 6장 표에 추가 → 9장 체크리스트로 UX 적용.**
