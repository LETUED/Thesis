# THESIS 페이지 구성 / 정보구조 (IA) + UX 표현 원칙

> 본 문서는 검증된 코드(`frontend/app|components|lib/*`, `backend/app/*`)와 교차검증해 작성했다.
> `✓` = 현재 코드에 존재(검증됨) / `*` = 신규 추가 / 상태 = 완료·다음·미래.
> Top-Down 철학: **사이트맵 순서 = 네비 순서 = 페이지 내 섹션 순서**로 3중 강제.

## 1. 사이트맵 (트리)

라우트 그룹은 Next.js App Router `(group)` 폴더 컨벤션 가정.

```
THESIS
│
├─ (marketing)  ── 공개 영역 (미인증 접근, SEO 인덱싱, 테마 토글 노출)
│   ├─ /                       랜딩(히어로·Top-Down 흐름·요금 미리보기·CTA)   * (기존 즉시 redirect → 공개 랜딩 전환)
│   ├─ /pricing                요금/플랜 비교 + 소개                          *
│   ├─ /learn                  콘텐츠(SEO) 허브 — 지표사전/국면해설/가이드     * 미래
│   ├─ /learn/[slug]           콘텐츠 상세 (광고는 여기에만)                  * 미래
│   ├─ /terms                  이용약관                                       * 다음(치명)
│   ├─ /privacy                개인정보처리방침                               * 다음(치명)
│   └─ /disclaimer             투자조언 아님 정식 고지(자본시장법 오인 방지)  * 다음(치명·우선)
│
├─ (auth)  ── 인증 (미인증 진입점)
│   ├─ /login                  로그인                                         ✓ 완료
│   ├─ /signup                 회원가입 (+ 약관·면책 동의 체크박스)            ✓ 완료 / 동의 게이팅 다음
│   └─ /auth/callback          매직링크/OAuth 콜백 (route.ts, UI 없음)        ✓ 완료
│
├─ (app)  ── 보호 영역 (미들웨어 세션가드, Top-Down 네비)
│   ├─ /dashboard              ① 매크로 국면 → ② 자산배분 → ③ 과신방지        ✓ 완료 (배너 위치 보정 필요)
│   ├─ /indicators             지표 상세(국면 근거 전체) — Pro 핵심           * 다음 (getIndicators 토큰화 연결)
│   ├─ /screener               종목 스크리너 (5단계)                          * 다음 (Pro/Quant)
│   ├─ /portfolio              포트폴리오 추적 (5단계)                        * 다음 (Pro)
│   ├─ /company/[symbol]       기업 상세 분석 (L4)                            * 미래 (Pro)
│   └─ /settings               설정 (테마·구독·로그아웃·알림)                 * 다음
│
├─ (billing)  ── 결제 결과 (정적 안내, 미보호 — 정보노출 위험 낮음)
│   ├─ /billing/success        구독 시작 안내                                 ✓ 완료
│   ├─ /billing/cancel         결제 미진행 안내(무압박)                       ✓ 완료
│   └─ /billing/manage         Stripe 고객포털 진입                           ✓ 완료
│
└─ 시스템
    ├─ /not-found              404 (중립톤)                                   * 다음
    └─ /error                  전역 에러 바운더리                             * 다음
```

## 2. 글로벌 네비게이션 (Top-Down 고정)

### 2-1. 공개(마케팅) 헤더 — `(marketing)`/`(auth)`
| 위치 | 항목 | 동작 |
|------|------|------|
| 좌 | THESIS 로고 | `/` 이동 |
| 중 | 제품 / 요금 / 학습 | `/#features` · `/pricing` · `/learn` |
| 우 | 테마 토글 · 로그인 · **무료로 시작** | 토글, `/login`, `/signup`(primary CTA) |

### 2-2. 앱(보호) 네비 — `(app)` · 순서 절대 재배열 금지
좌측 사이드바(데스크톱) / 하단 탭바(모바일).

| 순번 | 라벨 | 라우트 | tier | Top-Down 레이어 |
|------|------|--------|------|-----------------|
| 1 | 대시보드 | `/dashboard` | Free | 국면 결론(L1~L3 종합) |
| 2 | 지표 상세 | `/indicators` | Free 진입·Pro 전체 | 매크로 근거(L1·L2·L3) |
| 3 | 스크리너 | `/screener` | Pro/Quant | 기업 필터(L4) |
| 4 | 기업 분석 | `/company/[symbol]` | Pro | 기업 상세(L4) |
| 5 | 포트폴리오 | `/portfolio` | Pro | 리스크관리(L5) |
| — | 설정 | `/settings` | 전체 | 테마·구독·로그아웃 |

앱 헤더 우측: tier 배지(Free/Pro) + (Free) **Pro 업그레이드** / (Pro) 구독관리 + 테마 토글 + 아바타 메뉴(설정·**로그아웃**). 로그아웃 UI는 현재 부재 → 설정/아바타 메뉴에 신규 배치.

## 3. 인증 / 온보딩 / 세션 흐름

### 3-1. 인증 흐름 (검증된 현재 동작)
```
회원가입(/signup) ──signUp+emailRedirectTo──▶ 메일확인 ──code──▶ /auth/callback
   │                                                              │ exchangeCodeForSession
   │                                                              ▼
   └─ [신규] 약관·면책 동의 체크박스 필수                    세션 발급 → /dashboard
                                                                  │
로그인(/login) ──signInWithPassword──▶ redirectedFrom(기본 /dashboard) + router.refresh()
                                                                  │
미들웨어 updateSession ──미인증 시──▶ /login?redirectedFrom=path  (PROTECTED_PREFIXES=['/dashboard'])
```
- tier는 **백엔드 응답(`regime.tier`)에서만** 도출(클라이언트 위조 불가). 프론트는 토큰만 전달.
- profiles 자동생성(`handle_new_user` 트리거)과 자가승급 차단(RLS)은 **0001 마이그레이션 적용이 전제** → 미적용 시 가입 흐름이 런타임에서 깨질 수 있음.

### 3-2. 최초 로그인 온보딩 [신규·다음]
입문자 보호 위해 최초 로그인 시 3스텝 안내(localStorage로 1회만 표시):
| 스텝 | 내용 |
|------|------|
| 1 | 국면 카드 읽는 법 — "리스크온/중립/리스크오프는 방향일 뿐, 매수·매도 신호가 아닙니다" |
| 2 | 감정 슬라이더 사용법 — "표준편차가 아니라 '얼마나 흔들려도 견딜 수 있는지'를 고르세요" |
| 3 | 결론 무료 / 근거 유료 — "결론은 무료, '왜 그런지' 상세 근거는 Pro에서 펼쳐집니다" |

> 현재 대시보드 첫 진입 시 `AllocationPanel`은 result=null(아직 '근거 살펴보기' 미클릭)이라 도넛이 안 보인다 → 온보딩과 별개로 **빈 상태 안내 카피** 필요(아래 3-7).

### 3-3. 세션 만료 처리 [신규·다음]
- 대시보드는 서버에서 `getSession` access_token으로 프리페치하는데, **토큰 만료 시 백엔드가 익명 free로 폴백** → Pro 사용자가 갑자기 Free 화면(evidence 잠김)을 본다.
- 감지: 프론트에서 **토큰 존재 vs tier=free 불일치** 또는 getSession null을 감지 → 중립 배너 노출: *"세션이 만료되어 일부 기능이 제한됩니다 — 다시 로그인해 주세요."* (단정·공포 톤 금지)

## 4. 페이지별 상세 구성

> 표 열: [라우트 | 목적 | 섹션/컴포넌트 | 상태 | tier | 테마·반응형]
> 상태값 = 로딩/빈/에러/미인증/세션만료/Free/Pro. 반응형 = M(<640)/T(640~1024)/D(>1024).

### 4-1. 랜딩 `/` *(기존 redirect → 공개 랜딩 전환)*
| 항목 | 내용 |
|------|------|
| 목적 | 로그인 전 가치 전달·SEO 진입·가입 전환. "근거가 있습니다" 톤 |
| 섹션/컴포넌트 | ① 히어로(헤드라인+`무료로 시작`/`데모 보기`) ② Top-Down 흐름 시각화(매크로→자산배분→기업분석 3스텝) ③ 가치제안 3카드(한국특화·결론먼저·과신방지) ④ 국면 미리보기(읽기전용, 익명=free 마스킹) ⑤ 요금 미리보기(3플랜→`/pricing`) ⑥ FAQ ⑦ 풋터 disclaimer |
| 상태 | 로딩(미리보기 스켈레톤) / 에러(미리보기 섹션만 ErrorState, 페이지 유지) / 로그인 사용자는 헤더 CTA만 `대시보드로` 교체(강제 redirect 금지) |
| tier | 없음(공개). 미리보기는 익명 free 응답(이미 마스킹·EvidenceLocked) |
| 테마·반응형 | 토글 노출. M:1열, D:히어로 2열·카드 3열 |
| 비고 | 미리보기 SSR 호출 시 `API_BASE_URL`(localhost 의존) 명시 필요 |

### 4-2. 가격/소개 `/pricing` *
| 항목 | 내용 |
|------|------|
| 목적 | Free/Pro/Quant 비교·전환. "결론 무료·근거 유료" 설명 |
| 섹션/컴포넌트 | ① 플랜 비교표(₩0/₩9,900/₩29,900, 기능 매트릭스) ② CTA(Free=`무료로 시작`, Pro=`Pro 시작하기`, Quant=`준비 중`/대기등록) ③ 환불·해지 안내 ④ FAQ |

| tier | 표시는 공개. 결제 버튼은 세션 확인 후 백엔드 Checkout 위임 |

### 4-3. 콘텐츠 허브/상세 `/learn`·`/learn/[slug]` * (미래)
| 항목 | 내용 |
|------|------|
| 목적 | SEO 유입(지표사전 21개·국면 해설). 광고는 **콘텐츠 페이지에만** |

| disclaimer | 3중(layout 풋터 + regime 카드 인라인 + OverconfidenceBanner) 유지 |

### 4-8. 지표 상세 `/indicators` * (다음 — getIndicators 토큰화 연결)
| 항목 | 내용 |
|------|------|
| 목적 | 국면 결론의 "왜"를 전부 보여주는 Pro 핵심. L1→L2→L3 순(Top-Down, L2 한국 강조) |
| 섹션 | 레이어 탭/섹션(L2 한국 최상단 강조) · 티커 카드(change_pct·status·추세 라벨) · Pro면 latest/prev_close/SMA5·20·임계값 비교 · 외국인수급 카드(stub→"준비중", 가짜숫자 금지) · partial/failed 가시화 |
| 상태 | 로딩(카드 스켈레톤) / 빈(레이어 결측) / 에러(502→ErrorState) / 미인증→로그인 / **실패 티커: 회색 카드 + "일시 지연"** / **stale: 상단 "데이터 갱신 지연" 배지** / Free=요약(마스킹) / Pro=전체 |

## 6. 기능별 UX / 표현 원칙

> 모든 규칙은 검증된 코드 패턴(`RegimeSignalCard`·`AllocationPanel`·`AllocationDonut`·`EvidenceLocked`·`OverconfidenceBanner`·`globals.css`·`models.py`·`config.py`)을 기준으로 한다. [신규]는 미구현 제안.

### 6-0. 전역 표현 원칙
| # | 원칙 | 화면 규칙 |
|---|------|----------|
| P1 | "사세요" 금지 | CTA에 '매수/매도/사세요/파세요' 동사 금지 → '근거 살펴보기', '자세히 보기'만 |
| P2 | Top-Down 순서 | 화면 세로 순서·네비 = 국면→자산배분→기업분석, 절대 뒤섞지 않음 |
| P3 | 결론 먼저, raw 숨김 | 입문자 화면에 raw 수치(DXY/VIX 숫자·표준편차·%p) 직접 노출 금지 |

| P5 | 확률 표현 + stale 고지 | confidence는 % 비노출, 막대 3칸 + 확률 라벨. **데이터 stale이면 "갱신 지연" 고지** |

### 6-1. 국면 결론 (Regime)
화면 구조: 라벨 배지 → 감정 헤드라인 → 확률 confidence(막대) → top_drivers 칩 → evidence(잠금/해제) → disclaimer.

- **라벨**: `리스크온 / 중립 / 리스크오프` 3종만(`models.py`). '매도' 라벨 신설 금지 — `리스크오프`가 최고 경계.
- **색상**(감정 자극 최소화): 리스크온=emerald(`--regime-on`), 중립=slate, 리스크오프=**amber**(`--regime-off`), 강경계만 rose(`--regime-off-strong`). 빨강 가득 패닉 UI 금지.
- **헤드라인**: 백엔드 `_headline()` 그대로(관찰 톤 "~근거가 많습니다 / ~높아지고 있습니다"). 프론트 재작성 금지.
- **confidence**: weak=1/moderate=2/strong=3 막대 + 확률 라벨("비교적 뚜렷한 신호로 보입니다(확정 아님)"). 숫자/시그마 금지. `rationale`은 **Pro 전용**.
- **top_drivers**: 방향 칩만("원달러 약세 압력"), 칩 안 숫자 금지. coverage<0.5면 "데이터가 부족해 판단을 보류합니다(참고 수준)".

### 6-2. evidence 잠금/해제 (Free ↔ Pro 경계 — 전환 포인트)
| 항목 | Free (잠금) | Pro (해제) |
|------|------------|-----------|
| 테두리 | 점선(`border-dashed`) | 실선 |
| 아이콘 | 자물쇠(회색) | 없음 |
| 내용 | preview 1줄 + CTA "Pro로 자세히 보기" | comparison_text/근거 리스트 |
| 수치 | 0(방향만) | raw_value + 임계값 |
| 톤 | "근거가 더 있습니다"(압박 금지) | "임계값 초과"(명령 아님) |

> **[철학3 보강 — 필수]** 현재 Pro 해제 시 comparison_text("원달러 1,420원 — 임계값 1,400 초과")의 raw 수치를 **펼침 없이 즉시 노출**한다. 철학3은 'Pro 입문자'를 면제하지 않으므로 **Pro evidence도 기본 접힘 + "근거 수치 보기" 토글**로 보호한다. ('Pro라서 예외'는 철학3의 자의적 축소 해석)

### 6-3. 자산배분 (감정 슬라이더 + 도넛)
- **입력 슬라이더(5단계, 표준편차 0 노출)**: very_conservative "원금을 지키는 게 가장 마음 편합니다" ~ very_aggressive "큰 기회를 위해 큰 변동도 견딥니다"(`emotion.ts`). "변동성 15%" 같은 숫자 라벨 금지.
- **출력 도넛**: 주식=emerald/현금=slate/안전=amber, `isAnimationActive=false`. **[보정] 도넛 중앙엔 짧은 라벨(emotion.short)만**, 긴 `risk_label_text`(최대 25자+괄호)는 **도넛 아래 별도 줄**로(144px 원 중앙 넘침 방지). 범례에만 % 표기. 비율 합 100%·현금 최소 20% 불변식.
- **섹터 칩**: direction(overweight/neutral/underweight) "비중 확대 고려" 톤. 목표 비중 %·종목명은 **Pro evidence에서만**.

### 6-4. 한국특화 패널 [신규]
- 3개 지표(원달러·코스피·외국인수급)를 **한국 묶음으로 최상단** 강조(L2 가중 1.6).
- **표현**: Free는 16티커 전부 change_pct·추세 라벨만(raw는 Pro). 원달러 임계 톤 "주의 임계값 근접"(위험확정 금지).
- **외국인수급 — '준비중' 정직 표시(핵심)**: 백엔드 `available=False`/`source='stub'`. **0이나 가짜 그래프 절대 금지.** 회색 placeholder + "준비중" 배지 + "데이터 제공처 연동 후 제공 예정". regime/allocation 근거에도 "외국인 연속 순매도: 미수집(가중치 제외)" 명시. **Pro에서도 가짜 숫자 안 보임** — 철학8·정직성 차별점.

### 6-5. 과신방지 / disclaimer (3중 노출)

> **[보정]** 코드상 OverconfidenceBanner가 대시보드 **최하단**에 위치(dashboard/page.tsx:73) → 입문자가 결론 먼저 보고 한참 스크롤 후 봄. **상단 또는 ① RegimeSignalCard 직후로 이동**. `Disclaimer.is_advice=Literal[False]` 고정 — "투자조언" 표현 금지. 경고 임계값 도달해도 "리스크가 높아졌습니다"까지만.

### 6-6. 입문자(Free) vs 중급자(Pro) 표현 차이
| 차원 | Free | Pro |
|------|------|-----|
| 국면 | 라벨+감정 헤드라인+막대 confidence | + score[-100,+100], coverage, 지표별 comparison_text(접힘 토글) |
| 수치 | **change_pct·추세 라벨만(16티커 전부)** | latest/prev_close/sma5/sma20 전체 |
| confidence | 막대3칸+확률라벨 | + rationale("신호강도 X%, 커버리지 Y%…") |
| 자산배분 근거 | EvidenceLocked CTA | regime_reasons + korea_signals + constraint_notes |
| 임계값 비교 | 숨김 | "원달러 1,420원 — 임계값 1,400 초과"(접힘) |

> **핵심**: Pro라고 톤이 바뀌지 않는다. raw 수치를 보여줄 뿐 **여전히 '매도' 동사·행동 명령 없음**. Quant(₩29,900)는 tier 미정의 → 4곳 확장 후 동일 게이팅 패턴 재사용.

### 6-8. 접근성(a11y) [신규·다음]
- confidence 막대: `aria-hidden`만 두지 말고 **sr-only 텍스트로 probabilistic_label 재진술**(막대 의미 손실 방지).
- 감정 슬라이더(input type=range): `aria-label`에 더해 **`aria-valuetext={risk.headline}`**(현재 단계 감정 문구) — SR 사용자가 위치 인지.
- 도넛: 색상만으로 자산군 구분 금지 → **명도 차이/패턴 + 범례 텍스트** 병행.
- 색상 대비: **WCAG AA 검증**(특히 흰 배경 위 amber `--regime-off`). regime.ts/globals.css 토큰 대비비 측정.

### 6-9. 다크/라이트 토큰 방침 [신규·다음]
- 현재 `globals.css`에 **`:root` 라이트 토큰만** 존재, `.dark` 변형·토글 부재.
- **시맨틱 토큰 유지**: 컴포넌트는 `hsl(var(--regime-on))` 등 변수만 참조 → `.dark` 블록에 변수 재정의만 하면 **컴포넌트 수정 0**.
- 감정 자극 최소화 다크에서도 유지(리스크오프 amber 기조, rose 강경계만, 채도 과상승 금지).
- 토글: 시스템 우선(`prefers-color-scheme`) + 수동(localStorage). **라이트 디폴트 유지**(입문자 친화). 토글 위치는 헤더·설정. 차트(Recharts)는 변수 참조이므로 자동 대응.
```css
.dark {
  --background: 222 47% 8%;  --foreground: 210 40% 96%;
  --card: 222 47% 11%;       --muted: 217 33% 17%;
  --border: 217 33% 20%;
  --regime-on: 152 55% 50%;  --regime-neutral: 215 20% 65%;
  --regime-off: 38 92% 55%;  --regime-off-strong: 350 80% 60%;  /* amber 유지 */
}
```
