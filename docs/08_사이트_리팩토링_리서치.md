# 08. 사이트 대규모 리팩토링 — 딥리서치 종합 + 실행안

> 출처: 검증된 딥리서치(5각 · 25소스 fetch · 118 claim → 19 confirmed / 6 refuted).
> 문제의식: "탭마다 정보만 띡띡 보여줘서 단조롭다" → 대규모 리팩토링 검토.

## 한 줄 진단
"flat"의 정체는 셋이다 — ①**결론 없이 데이터만** ②**비교 기준 없음**("compared to what?") ③**돌아올 이유(루틴) 없음**.
리서치는 THESIS의 **"결론 먼저 / 근거는 펼쳐보기" 철학이 정답임을 검증**했다(progressive disclosure = NN/G가 학습성·효율·오류율을 개선한다고 명시; Seeking Alpha·Toss가 실제 제품으로 증명). **고칠 것은 철학이 아니라 실행이다.**

---

## 1. 구조 — flat → 층위(layered)
- **3층 노출**(progressive disclosure): glance(핵심 KPI ~2초) / detail(1클릭) / config(숨김). 핵심 KPI는 위·왼쪽.
- **conclusion-first 카드**: 결론(라벨/등급)을 먼저, 근거는 **눈에 보이는 "펼쳐보기"**로. 단 — 펼침 어포던스는 **반드시 보이게**(discoverability), 깊이 **≤2단계**, **위험경고/자주 쓰는 필터는 절대 숨기지 말 것**.
- **Seeking Alpha 패턴**: 100+ 지표를 단일 평결 + 차원별 A~F 등급으로 압축, raw는 1클릭. → THESIS는 **평평한 지표 표를 "차원별 등급 칩(정상/주의/위험)"으로** 바꾸고, 칩을 누르면 raw 수치 + 기준이 펼쳐지게.
- **Home/"오늘" 탭 신설**: "지금 국면?"에 2초 안에 답하는 허브 + 각 탭 진입. (현재는 탭별로 분절되어 흐름이 끊김 — Toss가 지적한 "모든 경험이 하나처럼 물 흐르듯"의 반대)

## 2. "살아있게" — 비교 맥락(compared to what?)
- 모든 숫자에 **비교 기준(cognitive landmark)**: 원/달러 1,420 → **1,400 주의 / 1,500 위험 선 대비 + 전기 대비 delta**; VIX 27 → **25 위험선 대비**. 우리 임계값을 "랜드마크"로 쓰면 raw를 안 까도 살아난다.
- delta/색 강조는 **절제**(과부하는 문서화된 안티패턴).
- **용어 툴팁**(DXY·VIX·외국인 수급): "입문자에게 수치 직접 노출 금지" 룰과 정합. ⚠️ **hover 전용 금지 → tap/focus 접근 가능하게**(WCAG 1.4.13, 모바일).

## 3. 돌아올 이유 — engagement = 행동 비계(scaffolding), not 정보
- **핵심 인사이트(Toss UX 리서치)**: 성공 투자자와 입문자를 가르는 건 더 나은 분석이 아니라 **일관된 반복 루틴**. → 정보를 더 주지 말고 **루틴 장치**를 줘라.
- **임계값 알림**: 원/달러>1,400 · VIX>25 · 외국인 10일 연속 순매도 → 카드/푸시. ("팔아라" 금지, "이런 근거가 있습니다" 톤).
- **가벼운 관심종목 + 자동 "최근 본 종목"**(수동 등록 안 해도 캡처).
- 저장한 **"내 규칙"**을 다시 보게 만들기.
- **온보딩 = 기능 나열 X → 따뜻한 "편지" + 실제 유저 목소리 인용**으로 신뢰 구축(돈 앱은 UI 변화에 민감). THESIS 톤("사세요 금지/과신 방지")과 정합.

## 4. 유입 · 전환
- 검증된 핵심: **결론은 Free, 근거·실시간·심화는 Pro**(gate the evidence, not the conclusion) — 이미 우리 모델. Seeking Alpha의 Free=컨센서스 / Pro=상세등급과 동일.
- ⚠️ **리서치 갭**: 구체적 페이월/티저/가격페이지/트라이얼 전술은 검증 통과한 게 없음 → 필요하면 별도 딥리서치.

## 5. 톤 · 비주얼
- **Robinhood**: 친근·비위협·잘난척 안 함 톤(입문자 심리 장벽↓). 단 — "Robinhood = 미니멀리즘"은 **기각(0-3)**이라 인용 말 것; **게임화(컨페티 등)는 규제 대상 + 과신방지 철학 위배 → 금지**(따뜻함만 빌리고 거래 유도 장치는 버린다).
- ⚠️ **리서치 갭**: 2025–26 모션/마이크로인터랙션/"프리미엄 느낌"은 얇게만 검증됨.

## 활성화 지표(instrument)
- **day-7 리텐션 ≥7% = 상위 25%**(Amplitude). 활성화 이벤트 1개 정의(예: "신호 근거 펼침" / "첫 알림 설정" / "첫 조립분석 완성") → 3-test(활성 유저가 더 잘 남나 / 세그먼트 불문 / 유도하면 전환↑)로 검증 후 북극성 지표로.

## 우선순위 로드맵 (영향 큼 · 비용 낮음 順)
1. **"compared to what?" 시그널 스트립 + 용어 툴팁** (대시보드 상단) — 임계값 보유, 백엔드 소폭(지표별 status 노출) 필요.
2. **Home/"오늘" 탭** — 2초 국면 + 허브. 프론트 위주.
3. **임계값 알림 카드** — settings 임계값 + snapshot에서 도출.
4. **conclusion-first 등급 칩**(지표·기업) — 프론트.
5. **관심종목 / 자동 최근본 / 알림 저장**(Supabase 테이블) → 루틴 = 리텐션. 백엔드.
6. **온보딩 "편지" + 이메일 다이제스트** — 후속.

## 피할 것 (딥리서치에서 기각 — 인용/적용 금지)
- F/Z 패턴 근거로 배치 정당화 (배치는 위/왼쪽 맞지만 F패턴 기전은 거짓 0-3)
- "Robinhood = 미니멀리즘/anti-data-dump" (0-3)
- "핀테크 자산앱 리텐션 93–96%" (0-3) / "2주 내 98% churn" (0-3) 통계
- "Density Disjoint" 명명 안티패턴 (1-2) / PD 4기법 분류 (0-3)
- hover 전용 노출(접근성) · 색/delta 과용 · 위험경고 숨기기
