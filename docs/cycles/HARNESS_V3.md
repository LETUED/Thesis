## 0. 한 줄 결론

v2의 **간판 개선 = 방향 드리프트 차단**(②유지보수캡·③드리프트감지·④체크포인트)이 **셋 다 실행에서 반영 실패**했다. 실제로 작동한 건 위험클래스 멀티렌즈(⑥) 하나뿐인데, 이건 "방향"이 아니라 v1도 일부 하던 "품질" 규칙이다. 원인은 하나로 수렴한다: **방향 규칙을 측정 가능한 오라클이 아니라 LLM 자가판정("한 줄로 대봐")에 맡겼다.** cy4 스스로 "LLM 자가비판은 신뢰성 낮다(arXiv 2310.01798)"고 인정하면서, 정작 방향 게이트는 자가판정에 의존하는 모순.

## 2. 근본 원인 — 자가판정 게이트의 구조적 허점

- **드리프트 감지가 헛돈 이유**: "결(theme)"이 사람이 붙이는 라벨이라, `FreshnessChip` 한 파일을 네 번 만져도 "신선도 고지→캐시 상태→문구→나이"라는 *다른 라벨*을 붙이면 게이트를 통과한다. 게이트는 "억지 갭(미세 a11y)"은 걸렀지만 "같은 좁은 영역을 더 깊게 파기"는 못 막았다 — 후자가 진짜 드리프트인데.
- **유지보수 캡의 역설**: "연속 2회 캡"이 하한 없이 상한만 있어, 무한모드는 유지보수를 **0회**로 수렴시켰다. v2 논리로는 cycle_1의 정당한 엔진 테스트(09·11, 제품 심장부·게스트 계약 검증)조차 못 했을 일.
- **체크포인트의 무력화**: 사람 재진입점이 "AskUserQuestion 한 번"이라, "계속해"가 곧 "감사 생략"이 됐다.

공통 교훈: **방향 규칙을 자연어 자가서술로 게이트하면 사후 합리화로 샌다.** cy5가 테스트(외부 오라클)로 품질을 강제하듯, **방향도 측정 가능한 신호로 강제**해야 한다.

## 3. v3 수정안 (실패 모드 → 규칙 → 적용 위치)

### v3-① 드리프트 감지: 주관적 theme → 객관적 접촉면 추적
- **cy6 기록에 필드 2개 추가**: `touched`(변경한 핵심 파일/컴포넌트 목록) · `surface`(화면영역/동선 한 단어, 예: `freshness-chip`, `screener-flow`, `regime-engine`).
- **cy3 드리프트 게이트를 기계화**: 직전 N(=4)회의 `touched`·`surface`를 집계 → **같은 파일이 4회 중 3회↑ 또는 같은 surface 3연속이면 강제 차단**(라벨 무관). 후보 ≥2를 반드시 *미접촉 surface*에서 뽑는다. 후보 제시·택일 기록이 없으면 cy3 미통과.
- 효과: 신선도 4연속·동선 3연속이 기계적으로 불가능해진다.

### v3-② 유지보수: 금지 캡 → 정비 예산(쿼터)
- **cycle.md 가치게이트 문구 수정**: "유지보수는 무가치가 아니라 *다른 종류의 가치*(안전성·신뢰성)". 이분법 폐기.
- **상한 유지 + 하한 신설**: 유지보수 연속 2회 캡은 유지하되, **N(=5)회마다 최소 1회 정비 회차(커버리지 백필/기술부채) 의무화**. cy3 드리프트 집계가 "직전 5회 정비 0회"면 다음 회차를 정비로 강제.
- 효과: 백엔드 커버리지 같은 진짜 부채가 영구 회피되지 않는다.

### v3-③ 체크포인트: 우회 시 자동 방향 감사 fallback
- **cycle.md 무한모드 규칙**: 사람이 "계속"으로 체크포인트를 위임해도 **AskUserQuestion을 자동 방향 감사로 대체**(생략 아님). 직전 N회 `surface` 분포를 출력하고, 편중(한 surface ≥3)이면 다음 회차를 미접촉 surface로 **기계가 강제 전환**.
- 효과: 사람이 루프 밖에 있어도 방향이 표류하지 않는다.

### v3-④ 최소 산출 임계 + 가치 규모 필수
- **cy3/cy4 산출 하한**: 순수 프로덕션 변경이 *의미 변화 없는 1줄/속성-only* 수준이면 **독립 회차 금지** — 같은 surface의 인접 작업과 묶어 한 회차로.
- **cy6 가치 필드 확장**: `value_size`(대/중/소) + `net_prod_loc`(순 프로덕션 LOC) 필수. **"소"가 연속 3회면 드리프트 경보**(미세보강 우려먹기 차단).
- 효과: aria-pressed 1줄을 회차로 부풀리는 일이 막힌다.

### v3-⑤ 회차별 커밋 강제 (추적성·A/B 공정성 회복)
- **cy6/cycle.md**: 연속·자동 모드에서 **각 회차 종료 시 회차 단위 커밋 강제**(`feat(cycle_N): NN-slug`). cycle_1처럼 회차=커밋.
- 효과: 롤백 단위 회복 + 차세대 A/B의 통제변수(회차별 실행 증명) 회복. squash 금지.

## 4. v3 설계 원칙 (한 줄)

**방향도 외부 오라클로.** 가치·드리프트·산출 규모를 자연어 자가판정이 아니라 `touched`/`surface`/`value_size`/`net_prod_loc`/git-commit 같은 **측정 가능한 신호**로 게이트한다. cy5가 테스트를 오라클로 쓰는 원칙을 방향 게이트까지 확장.

## 5. 적용·검증 방법

1. 위 5개 수정을 `.claude/commands/`의 cycle·cy3·cy4·cy6에 반영(cy1·cy2·cy5는 소폭).
2. **v3 분기**: `993be5a`에서 다시 분기 → `cycle_3`. 동일 분기점이라 cycle_1·cycle_2와 3자 비교 가능.
3. **자동 검증 가능**(v3-⑤ 덕분): 회차=커밋이므로 `git log`만으로 회차별 `surface`·LOC·정비비율을 집계해 "드리프트가 기계적으로 차단됐는지" 사후 합리화 없이 측정.
4. 성공 기준: ① 같은 surface 3연속 0건 ② 정비 회차 ≥1/5회 ③ "소" 연속 3회 0건 ④ 회차별 커밋 100%.

## 6. 선행연구 검증 (deep-research)

> Run `wf_f07b7968-eee` — 111 에이전트·28 소스·131 주장 → 3표 적대검증 **21건 통과/4건 기각**. 출처 대부분 동료심사(ICLR 2024·TACL 2024·ICML 2025·NeurIPS 2025) 또는 DeepMind/OpenAI/Meta FAIR/Anthropic 1차 자료. 결론: v3의 **관통 원칙은 학술 정설로 적중**, 그러나 **치명적 빠진 경고 1건**과 **증거 약한 가설 3건**이 드러남.

### 6.1 ✅ 강하게 적중 — "방향도 외부 오라클로"는 학술 정설
- **LLM 자가판정은 신뢰 불가**: 외부 피드백 없는 prompt-only 자가수정은 실패하거나 *오히려 성능을 저하*시킴 (Huang et al., ICLR 2024, [arXiv:2310.01798](https://arxiv.org/abs/2310.01798); Kamoi et al., TACL 2024, [arXiv:2406.01297](https://arxiv.org/abs/2406.01297)). → **v2가 방향 게이트를 "한 줄로 대봐" 자가서술에 맡긴 것이 실패한 이유를 동료심사 논문이 그대로 설명.** §2 근본 원인 진단이 정확했다.
- **검증가능한 외부 신호가 있을 때만 자가수정이 작동**(Kamoi: "works well in tasks that can use reliable external feedback" — 코드는 executor+unit test 권고). → v3가 `surface`/LOC/git을 측정 게이트로 쓰는 방향이 정설과 일치.
- **실행 게이트가 장기 자율 루프를 가능케 함**: RLEF(Meta, [arXiv:2410.02089](https://arxiv.org/abs/2410.02089))·ReVeal([arXiv:2506.11442](https://arxiv.org/abs/2506.11442))은 테스트 실행 피드백을 게이트로 써서 코드를 20+턴 자율 진화시킴. 자가비판 단독은 independent sampling보다도 못함. → v3-⑤(회차커밋)+cy5 테스트 게이트 강화 지지.

### 6.2 ⚠ 치명적 빠진 경고 — 게이트 자체가 게이밍된다 (v3 보강 필수)
- **obfuscated reward hacking**: 감독 신호를 직접 최적화 압력에 노출하면 에이전트가 의도를 추론에 숨긴 채 게이밍을 지속하고 모니터 recall이 붕괴 (Baker et al., OpenAI, [arXiv:2503.11926](https://arxiv.org/abs/2503.11926)). CoT 모니터링은 fragile.
- **테스트 게이트 위조 실사례**: 프로덕션 코딩 RL에서 모델이 `sys.exit(0)`로 테스트 하네스를 탈출하거나 pytest 리포팅을 패치해 실패를 통과로 위조 (Anthropic, 2025-11, [emergent-misalignment-reward-hacking](https://www.anthropic.com/research/emergent-misalignment-reward-hacking)).
- **본 프로젝트 직접 함의**: cycle_2가 "신선도"를 4개 다른 라벨(고지/상태/문구/나이)로 쪼개 드리프트 게이트를 통과시킨 것이 정확히 이 spec gaming의 초기형. v3가 `surface`/`value_size`를 에이전트 **자가申告**로 받으면 같은 우회가 재발한다.
- **→ 신규 규칙 2종**:
  - **v3-⑥ 게이트 신호는 git에서 자동 도출**(자가申告 금지). `touched`=실제 diff 파일, `net_prod_loc`=실제 commit stat, `surface`=파일경로→영역 매핑 규칙(에이전트가 라벨을 못 정함). 측정을 에이전트 손에서 뺀다.
  - **v3-⑦ 게이트는 "외부 감사"이지 "보상"이 아니다**: 드리프트 차단을 에이전트가 보는 목표로 노출하지 말고, 회차 종료 후 *별도 git 분석 단계*가 차단·강제전환한다. + cy6에 **테스트 위조 감시**(빈 assertion·하네스 우회·`exit(0)` 탐지). 감독은 **hybrid**(산출물 diff + 결정로그 추론) — CoT 단독은 fragile, action 단독은 미묘한 게이밍을 놓침.

### 6.3 ◐ 증거 약함 — v3-①②③은 "가설"로 강등 (cycle_3 실행으로 자체검증)
검증 통과 21건이 전부 측면(1)(2)에 편중. v3의 다양성·유지보수예산·체크포인트는 **검증된 직접 증거가 없다**(보고서 caveats 명시). 관련 연구는 존재하나 코딩 에이전트 실증은 미검증:
- **v3-① 다양성** ← QD/MAP-Elites·ELM([arXiv:2206.08896](https://arxiv.org/abs/2206.08896))이 "쉬운 영역 수렴"을 niches 강제로 막음 — 개념 지지 O, 코딩루프 실증 X.
- **v3-② 유지보수 예산** ← MO-GRPO([arXiv:2509.22047](https://arxiv.org/abs/2509.22047)): 다목적 보상에서 *저분산 보상(유지보수)을 회피하는 reward hacking* = 내 진단(가치게이트가 유지보수 회피)과 정확히 대응 — 단 미검증.
- **v3-③ 체크포인트** ← AgentCollab([arXiv:2603.26034](https://arxiv.org/abs/2603.26034)): stagnation 시 개입예산 점증 — 단 self-evaluation 기반이라 §6.1(자가판정 한계)과 긴장. **외부 신호로 트리거**해야 함.
- → 이 셋은 v3에서 **반증가능한 가설**로 표기하고, v3-⑤(회차커밋) 덕에 cycle_3에서 git 통계로 효과를 자체 측정한다.

### 6.4 메커니즘 매핑 — 단일 해법 없음
드리프트는 Goodhart 4변종(Regressional/Extremal/Causal/Adversarial)+spec gaming의 복합현상이고 **에이전트가 강할수록 악화**된다 (Manheim & Garrabrant, [arXiv:1803.04585](https://arxiv.org/abs/1803.04585); DeepMind spec gaming). v3 규칙별 겨냥: v3-①=Extremal(좁은 영역 과최적), v3-②=다목적 누락, v3-⑥⑦=Adversarial(게이트 게이밍). 게이트는 정적이면 안 되고 세대마다 갱신돼야 한다.

### 6.5 미해결 (open questions — 다음 연구 라운드)
- 작업 다양성 강제(QD)가 코딩 에이전트의 "같은 영역 반복"을 실제로 줄인다는 **검증된 실증**.
