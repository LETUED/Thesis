# 15 · 스크리너 라벨 정직화 (screener-label-honesty)

- **작업**: dashboard ④ "종목 스크리너[준비중]" 죽은 카드와 Sidebar "준비중" 배지를, 실물(동작하는 기업분석 조회기)에 맞게 정직 재정의·연결.
- **브랜치**: `cycle_2` · **세대·회차**: cycle_2 / 15 (무한모드 15회차, 자율)

## ★ 사용자-가치
**동작하는 기업분석을 "준비중"으로 가려두고 잘못된 이름("종목 스크리너")으로 부르던 부정직을 바로잡아, 입문자가 실물에 맞는 이름·동선으로 진입한다.**

## 조사 (cy1, Explore) — A vs B, A 권고
- **라벨 3불일치 확정**: 실물/페이지제목="기업분석"(조회기) vs dashboard ④="종목 스크리너[준비중]+무링크" vs Sidebar="기업분석[준비중]". 정밀하게는 **배지 1곳 + 카드 1곳**이 손볼 지점.
- **A(정직 재정의) 권고**: 13·14가 이미 "다음 차례(#2)"로 지목한 **실재 부정직**(동작하는데 준비중+무링크) 닫기 → 억지 갭 아님, v2 정신 부합.
- **B(유지보수) 배제**: 12회차가 "억지 갭"이라 못박은 미세보강에 가까움 → 역행.

## 진행 (cy5) — 변경 파일 2
- `frontend/app/dashboard/page.tsx` — ④ 카드: 제목 "종목 스크리너"→**"기업분석"**(Building2), "준비중" 배지 제거, 본문 "기업을 검색해 PER·ROE 등 재무 신호를 바로 확인하세요", **`/screener` 링크**(익명→/signup, 인증→/screener — /lab 카드 동선 통일)
- `frontend/components/app-shell/Sidebar.tsx` — `/screener`의 `soon:true` 제거(동작하므로 배지 거짓). 라벨 "기업분석"은 이미 정직이라 유지.

**검증**: 타입체크 클린 · 테스트 **41/41**(게이트 회귀: /screener protected 유지) · 빌드 그린.
