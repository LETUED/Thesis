# 조립 분석(/lab) 캔버스 — '페이지' 레벨 옵션/기능 제안

> 대상: `/lab` 다중 페이지 노드 캔버스(React Flow). 재무 지표 블록을 기업 블록에 연결해 분석을 조립하는 화면.
> 평가 축: **가치 × 공수 × Pro 게이트 가능성 × 설계철학 적합**. 빠른 승리(Quick Win) 별도 표시.
> 데이터는 현재 **목업(MOCK)** — 추후 DART 연결. 외부 노출(공유/export)이 늘수록 '실데이터 아님' 고지가 더 중요.

## 2. 빠른 승리 (Quick Win) — 기존 코드 재사용, 공수 최소

착수 즉시 가치가 나오고 데이터 모델 변경이 0이거나 옵셔널인 항목.

1. **페이지 복제** — `serializeActive()` 결과 깊은복사 + `importPage`(BlockCanvas.tsx:394)의 `idMap` id 재매핑 로직을 그대로 재사용해 `pages`에 push. `canAddPage`(:99) 체크를 끼우면 Free 상한 페이월이 공짜로 따라온다.
2. **삭제 확인 / 실행취소** — `deletePage`(:251)는 현재 확인 없이 즉시 삭제. 토스트 '실행취소'(직전 `pages` 스냅샷 보관) 또는 최소 confirm 1단계. 데이터 손실 방지 = 신뢰.
3. **탭에 기업 칩 노출** — 탭 렌더(:540)에서 `COMPANIES`(data.ts:15)를 조회해 `p.companyId` → 티커 칩 추가. 모델 변경 0.
4. **마켓 공유 동선 단축** — 이미 페이지 단위 공유가 동작(`getCurrentPage={serializeActive}` :725). 탭 우클릭/⋯에 '마켓에 공유' 바로가기만.
5. **페이지 템플릿으로 시작** — `TEMPLATES`(data.ts:89) 데이터는 이미 존재. `addPage` 시 빈 페이지 대신 선택 → `blockIds`를 자동 좌표 배치 + company 연결. (공수는 자동 레이아웃만 중간)

## 3. Pro로 게이트해 BM 강화할 것

> **원칙:** 페이지 '수량'은 이미 페이월(`FREE_PAGE_LIMIT`). 추가 게이트는 **결과 산출물·고급 기능**에만. 꾸미기(색/아이콘)·안전장치(삭제확인/잠금)·이름(리네임)은 **Free 유지** — 인색해 보이면 '결론 무료' 철학 훼손.

| Pro 게이트 후보 | 게이트 방식 | 근거 |
|----------------|-----------|------|
| **이미지/PDF Export** | Free = 큰 워터마크·저해상도 / Pro = 클린 고해상도 | Miro 무료=저해상도/유료=고해상도 패턴 |
| **읽기전용 공유 링크** | 공유(생성)=Pro, 보기=무료(SEO 유입). 뷰어에서 Pro 블록 근거는 잠금 유지 | '결론무료/근거유료' 정합 + 5단계 콘텐츠 광고 연결 |
| **비교 모드** | 단독 Pro 기능(차별화). 단 MVP 후순위(5단계 이후) | 분석툴 핵심 차별화 |
| **'내 페이지를 템플릿으로 저장'** | 마켓 publish와 연계 → Pro | 기본 템플릿은 Free, 저장은 Pro |
| **고급 템플릿** | Pro 블록(PEG 등) 포함 템플릿은 자연 게이팅 | `peg` 블록 이미 `pro:true` (data.ts:74) |

**자연 게이트(코드 추가 불필요):** 복제·페이지 추가는 `FREE_PAGE_LIMIT`에 걸려 2번째부터 페이월 → 기능 자체를 막지 말고 **'저장/추가 시 페이월 재사용'**. 이미 `addPage`가 이 패턴(:239).

## 4. 피할 것 (과신 유발 / 스코프 크리프)

### 철학 충돌 — 반드시 방지
1. **페이지 색을 신호색(`regime-on`/`regime-off`)과 같은 팔레트로** → '초록=매수 / 빨강=매도'로 오독. 라벨용은 **신호색과 분리된 중립 팔레트**로 강제. (탭 활성색이 이미 `bg-regime-on/15` :563 — 라벨 색과 충돌 주의)
2. **비교 모드의 순위·승패 라벨**('A가 더 좋음', 1·2등 배지) → '사세요' 금지 철학 위반. **'근거 나열' 톤 유지**, 우열 단정 금지.
3. **Export 이미지·공유 뷰어에 disclaimer 누락** → 현재 결과 패널의 3중 고지를 캡처/뷰어에도 강제 삽입. **'매수권유 아님 · 목업 데이터'** 오버레이 필수. 실데이터(DART) 전환 전까지 공유/export에 **목업 배지 상시 노출**.

### 스코프 크리프 — 현 단계 보류
- **폴더/그룹(섹션):** 스키마 계층 추가 → `LabGraph`/마이그레이션 영향. 페이지 수십 개일 때만 ROI. CLAUDE.md '과도한 추상화 금지' 위반. n8n도 최근에야, Retool은 아직 요청 단계. **페이지 10개+ 사용자 나오면 그때.**
- **썸네일/미리보기 갤러리:** 스냅샷 생성·저장·무효화 비용. 페이지 적은 MVP에서 ROI 낮음. **Export(toPng) 먼저 깔면 그 캐시 재활용** → export 이후로.
- **페이지 잠금:** 1인 편집 SaaS라 협업 충돌이 없어 동기 약함. 후순위.
- **비교 모드:** 가치 높지만 ReactFlow 인스턴스 2벌·상태/성능 부담 큼. '복제 + 페이지별 기업'으로 유사 충족되므로 **5단계(스크리너/포트폴리오) 이후.**

### 구현 주의(충돌)
- 탭 드래그·복제 시 캔버스 `onDrop`의 `DRAG_KEY`(`application/thesis-block` :40)와 **다른 dataTransfer 타입** 사용 — 탭은 이미 `PAGE_DRAG_KEY`(:41)로 분리됨. 복제 신규 코드도 동일 원칙.
- 복제는 metric id를 `idRef`로 **새로 발급**해야 페이지 간 id 충돌 방지(`importPage`의 `idMap` 패턴 재사용 :398).
- 옵셔널 필드(`color?`/`icon?`/`locked?`) 추가 시 `normalize()`(persistence.ts:21)는 v2를 그대로 통과시키므로 하위호환 안전. **`any` 금지** — 옵셔널 타입으로.

## 6. 스키마 필드 (PageGraph 확장안)

현재 (`lib/lab/persistence.ts:4`):
```ts
export interface PageGraph {
  id: string;
  name: string;
  companyId: string;
  companyPos: { x: number; y: number };
  metrics: { id: string; blockId: string; x: number; y: number }[];
  edges: { id: string; source: string; target: string }[];
}
```

제안 확장 (모두 **옵셔널** → `normalize()` 하위호환 유지, `any` 금지):
```ts
export interface PageGraph {
  id: string;
  name: string;
  companyId: string;
  companyPos: { x: number; y: number };
  metrics: { id: string; blockId: string; x: number; y: number }[];
  edges: { id: string; source: string; target: string }[];

  // --- 확장(옵셔널) ---
  color?: PageColor;       // 색 라벨 — 신호색과 분리된 중립 팔레트만
  icon?: string;           // 이모지 1자(아이콘 라벨)
  locked?: boolean;        // 읽기전용(노드 이동/삭제 차단) — 후순위
  templateId?: string;     // 어떤 템플릿에서 시작했는지(분석용 메타)
  createdFrom?: string;    // 복제 출처 페이지 id(추적용, 선택)
}

// 신호색(regime-on/off)과 겹치지 않는 중립 팔레트로 한정
export type PageColor = "slate" | "violet" | "amber" | "sky" | "rose" | "teal";
```

마켓/공유용 메타(서버측 `lab_templates` 또는 별도 share row — `marketplace.ts:4` 인접):
```ts
export interface PageShare {
  id: string;          // 공유 토큰(URL)
  pageGraph: PageGraph;
  authorId: string;
  isMock: true;        // 실데이터(DART) 전환 전까지 항상 true → 목업 배지
  readOnly: true;      // 뷰어는 읽기전용, Pro 블록은 잠금 유지
  createdAt: string;
}
```
