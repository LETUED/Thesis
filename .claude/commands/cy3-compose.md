[사이클 3/6 · 구성] 조사 + 검색을 종합해 작업을 구조화한다. [v3: git 자동 드리프트 차단 + 최소 산출 임계]

작업: $ARGUMENTS

★ 방향 앵커 (cycle.md 방향 게이트의 cy3 재확인 — git 신호로, 자가서술 아님):
- **드리프트 자동 차단**: 직전 회차 커밋을 `git log --oneline --grep='cycle_3' -6` + `git show --stat <hash>`로 보고, 변경 파일 경로에서 surface를 도출한다. 같은 surface가 직전 3회 연속이면 그 surface는 이번 회차 금지 — 결이 다른(미접촉 surface) 후보 ≥2개를 제시한 뒤 택한다. 판정은 git diff 사실에 근거한다('다른 결이다'라는 선언만으로는 통과 불가).
- **정비 예산**: 직전 5회에 정비(테스트/기술부채/커버리지) 0회면 이번을 정비로. 정비 연속은 2회까지.
- **사용자-가치 한 줄 + value_size 잠정**: 이 작업이 진전시키는 가치를 한 줄로 명시하고 규모를 대/중/소로 잠정 표기(확정은 cy6에서 git LOC로). 못 대면 '유지보수: <무엇>'으로 분류·명시(숨기지 않는다).
- **최소 산출 임계**: 계획된 순 프로덕션 변경이 '의미변화 없는 1줄/속성-only'면 독립 회차로 만들지 말고 같은 surface 인접 작업과 묶는다. value_size 'small'이 직전 2회 연속이면 더 큰 가치를 택한다.

정리할 것:
- 요구사항 확정 — 무엇을 만들고, 무엇은 안 만드는가(범위 경계)
- 파일/모듈 단위 분해 — 어떤 파일을 새로 만들고 어떤 파일을 수정하나
- 엣지케이스·실패 경로 — 데이터 null, 캐시 stale, Free/Pro 게이팅 등
- 제약 체크 — 설계철학·임계값·플랜구조에 맞는가

중복생성 가드(자율 루프의 핵심 실패모드): 기존 코드·주석·노트를 '새로 만들 스펙'으로 오인하지 말 것. cy2 검색 결과의 재사용 대상을 먼저 확인하고, 이미 있는 것은 다시 만들지 않는다(중복 클래스·컴포넌트 생성 금지).

산출: 구조화된 작업 명세 + **사용자-가치 1줄**(또는 '유지보수: <무엇>') + value_size 잠정 + 드리프트 자동점검 결과(직전 surface 목록 + 이번 surface + 충돌 여부). 아직 구현하지 말 것.
