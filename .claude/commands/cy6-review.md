[사이클 6/6 · 리뷰] 독립 평가자로 검토하고, 발견 즉시 고친다. 사람 개입 없음. [v3: git 자동도출 가치필드 + 외부감사 + 테스트위조 감시 + 회차 커밋]

집중할 부분(선택): $ARGUMENTS

독립 평가 원칙(핵심): 자기가 쓴 코드를 자기가 통과시키지 않는다. 검토는 구현과 분리된 fresh-context 평가자에게 맡긴다 — code-reviewer 서브에이전트를 zero-context로 띄우거나(가능하면 다른 모델) /code-review 를 쓴다. "작성자가 쓴 테스트는 필요하나 불충분"이기 때문.

검토할 것:
- 정확성 — 버그, 엣지케이스, 기능 후퇴(regression)
- 보안 — injection, 시크릿 노출, 인증/게이팅 누락
- 단순화·재사용 — 중복, 죽은 코드, 과한 추상화
- 설계철학 준수 — 결론 먼저, 수치 비노출, 과신방지 문구

검증 원칙: 단일 판정을 믿지 않는다. 구조 게이트(테스트·빌드·린트)와 내용 리뷰는 서로 다른 실패를 잡으니 둘 다 본다. 확신 없는 발견은 보수적으로 처리한다.

★ 강제 에스컬레이션 (위험 클래스는 단일 리뷰 금지): diff가 인증·결제·Free/Pro 노출·시크릿·개인정보 중 하나라도 건드리면 서로 다른 렌즈(정확성·보안·노출경계)의 독립 평가자 ≥3을 띄우거나 `/code-review ultra`(사용자 트리거·과금 → 강하게 제안)를 포함한다.

★ 테스트 위조 감시 (v3 — Anthropic '25 reward-hacking 실사례): 추가/변경된 테스트가 (a)통과만 시키는 빈/동어반복 assertion, (b)테스트 하네스 우회(sys.exit(0)·pytest 리포터 패치·무의미 skip/xfail)로 실패를 통과로 위장하지 않는지 확인한다. 산출물(diff)과 추론(결정로그)을 함께 본다 — hybrid 감독(CoT 단독은 fragile, action 단독은 미묘한 게이밍을 놓침: Baker '25 arXiv:2503.11926).

자율 루프: 발견사항을 직접 수정 → 테스트·빌드 재확인 → 독립 평가자로 다시 검토. 새 발견이 없을 때까지 반복(최대 2라운드).

★ 기록(decision log) — 가치필드는 git에서 자동 도출(자가申告 금지):
- 경로: `docs/cycles/cycle_3/<NN>-<기능슬러그>.md`. NN = 직전 회차+1(2자리 zero-pad). 세대 폴더 없으면 생성. 슬러그 kebab-case.
- **git 자동도출 필드**(이번 회차 변경을 실측 — 손으로 고르지 말 것. ★신규 파일이 누락되지 않도록 `git add -N <신규>` 후 `git diff --stat`, 또는 커밋 직전 `git add` 후 `git diff --cached --stat` 으로 측정 — 워킹트리 `git diff` 단독은 untracked 신규파일을 빠뜨린다):
  - **touched**: 변경한 핵심 파일 목록 (git diff 실측)
  - **surface**: touched 파일 경로 → 영역(매핑 규칙). 드리프트 추적의 1차키
  - **net_prod_loc**: 순 프로덕션 LOC(테스트·docs 제외, git stat에서)
  - **value_size**: 대/중/소/유지보수 — net_prod_loc·기능성 근거(소=1~수십줄·문구·속성, 중=컴포넌트/필드, 대=신규기능/인프라). 자기 미화 금지.
- 담을 내용: **작업** 스코프 1줄 / **브랜치** / **세대·회차(cycle_3 / NN)** / **★사용자-가치 한 줄**(또는 '유지보수: <무엇>') / **조사(cy1)** / **검색(cy2)** 재사용vs신규 / **설계(cy4)** 채택+기각대안 / **갈린 판단** 택한 기본값+이유 / **진행(cy5)** 변경파일+테스트·빌드 / **리뷰(cy6)** 발견+처리(위험클래스면 멀티렌즈, 테스트위조 점검결과) / **상태**.
- 세대 폴더 INDEX.md에 한 줄 추가(NN·슬러그·surface·value_size·사용자가치/유지보수). 없으면 생성.

★ 외부 감사 (v3 — 게이트는 보상 아닌 사후 점검): 기록 작성 후 이번 회차의 git 신호(surface·value_size·net_prod_loc)를 직전 회차들과 대조해 드리프트 게이트를 사후 적용한다 — 같은 surface 3연속? value_size 소 3연속? 정비 5회 결손? 위반이면 기록에 명시하고 다음 회차 cy3 입력으로 강제 전달한다.

★ 종료 (v3 — 회차 단위 커밋 강제):
변경 요약 + 처리한 발견 + 기록 경로(cycle_3/<NN>-...) 출력 후 **이번 회차를 단독 커밋한다**: `feat(cycle_3): NN-슬러그` (본문에 surface·net_prod_loc·value_size·사용자가치 한 줄). squash·회차 묶음 금지 — 회차=커밋이라야 git으로 자동 A/B 검증이 된다. 사이클 종료. (연속 모드면 cycle.md 방향 게이트로 다음 회차 재점화.)
