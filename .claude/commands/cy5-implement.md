[사이클 5/6 · 진행] 설계대로 구현하고, 테스트·빌드가 통과할 때까지 스스로 고친다.

작업: $ARGUMENTS

규칙:
- 설계 범위만 — 요청 안 한 리팩토링·추상화 금지
- 가드레일: any 금지, 시크릿 하드코딩 금지, SQL/command injection·XSS 주의
- 라이브러리 호출: Next.js·FastAPI·Supabase·Stripe 등의 정확한 현재 문법이 불확실하면 context7 MCP로 문서를 확인하고 쓴다(존재하지 않는 API 환각 금지)
- 설계철학: 결론 먼저·근거는 펼쳐보기, 수치 비노출(Free), "지금 사세요" 금지, 과신방지 문구
- 주석은 WHY가 명확할 때만

기계 게이트 — 외부 오라클이 진짜 게이트다(LLM 자가점검만으론 품질 못 지킨다 — Huang ICLR'24, Kamoi TACL'24):
- 변경한 동작에 대한 테스트를 함께 작성하고, 그 테스트가 통과해야 task 완료로 본다. **단, 테스트는 의미 있는 assertion이어야 한다 — 통과만 시키는 빈/동어반복 테스트, 하네스 우회(sys.exit·리포터 패치·무의미 skip)는 금지(cy6 테스트 위조 감시 대상).**
- 빌드/타입체크 통과 필수(프론트 `npm run build`, 백엔드 import·`py` 점검).
- 실패 시 원인을 고쳐 재시도(최대 3회). 3회 후에도 실패하면 원인을 로그로 남기고 리뷰 단계에서 다룬다.

산출: 테스트·빌드 그린 코드 + 검증 결과. 사용자 승인 없이 /cy6 으로 진행. (커밋은 cy6 종료에서 회차 단위로 하므로, 변경을 흩뜨리지 말고 이번 회차로 묶을 준비를 한다.)
