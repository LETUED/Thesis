완료 강제는 내장 /goal 위에 얹는다(역할 분리 — "멈추지 마"는 /goal, "어떻게"는 /cycle):

- 그 뒤 6단계는 멈추지 않고 순서대로 실행하고 단계 사이 사용자 승인을 묻지 않는다 — "끝날 때까지 멈춤 금지"의 하드 게이트는 /goal Stop 훅이 강제하고, 이 명령은 그 위에서 '방법'만 책임진다.

★ 방향 게이트(cy1 전에 먼저 — v1 드리프트를 막는 핵심):
- **가치 게이트**: 이번 주제가 어떤 '북극성'을 진전시키는지 한 줄로 댄다. THESIS 북극성 = ② "유저가 부담없이 올 수 있는 시스템" + 설계철학(결론무료/근거유료·과신방지·한국특화). 한 줄로 못 대면 그 사이클은 '유지보수'(테스트·리팩토링·인프라)다.
- **유지보수 캡**: 유지보수 사이클은 연속 2회까지만 허용. 3회째는 반드시 사용자-가치 주제를 고른다(그린이라고 done 아님 — 가치가 done의 일부).
- **드리프트 체크**: 직전 3사이클(docs/cycles/cycle_<세대>/ 최신 회차)이 같은 결(예: 전부 'backend test')이면 멈추고, 결이 다른 후보 ≥2개를 제시한 뒤 택한다.
- **주기 체크포인트**(연속/무한 모드): N(기본 5)사이클마다 자동 재점화 대신 사람에게 방향 재확인(AskUserQuestion). "사람은 루프 밖에서 의도를 준다"의 재진입점 — 무한모드가 사람 없이 표류하지 않게.

방법 — 기계 게이트로 품질 보증(사람 승인 대체):
- 설계는 스스로 비판하고, 진행은 테스트·빌드 통과를 강제하며, 리뷰는 독립 평가자로 검증 후 즉시 수정한다.
- 시작 시 TaskCreate로 6단계 체크리스트 생성, 단계마다 in_progress/completed 갱신(가시성용 — 승인 요청 아님).
- 판단이 갈리면 가장 합리적인 기본값을 택하고 기록(decision log)에 남긴 뒤 계속 간다.

브라운필드 안전수칙(THESIS는 기존 코드베이스 — 선행연구가 가장 경고하는 영역):
- 한 사이클의 스코프는 '기능 1개'로 좁힌다(반복당 1개 작업 원칙). 큰 작업은 쪼개 여러 번 돌린다.
- 파괴적·광범위 변경이 예상되면 git worktree 격리에서 실행한다(샌드박스 = 권한스킵의 유일한 보안경계).
- 사람은 루프 '밖'에서 스펙·리뷰만 관리한다("sit on the loop, not in it"). 자율은 실행에 한정, 의도는 사람이 준다.

단계 — 각 단계는 해당 스킬을 Skill 도구로 실제 호출해 상세 지침을 적용한다(이름만 따르지 말고 그 단계의 cy 스킬을 로드해 그 안내대로 수행):
1. 조사 → cy1-investigate → 2. 검색 → cy2-search → 3. 구성 → cy3-compose → 4. 설계 → cy4-design → 5. 진행 → cy5-implement → 6. 리뷰 → cy6-review
각 스킬 호출 시 인자로 이번 작업 + 직전 단계 산출을 넘겨 맥락을 잇는다. (방향 게이트는 cy1 진입 직전에 한 번, cy3 구성에서 다시 확인한다.)

기계 게이트(사람 승인 대체):
- 설계(4) 후: 계획을 스스로 점검 — 범위·제약·타입·엣지케이스 누락 없는지. 문제 있으면 고쳐서 다음으로.
- 진행(5) 후: 테스트·빌드/타입체크가 통과해야 다음으로. 실패하면 고쳐 재시도(최대 3회), 그래도 안 되면 원인 로그 후 리뷰에서 처리.
- 리뷰(6): 독립 평가자(fresh-context 서브에이전트)로 검토 → 발견사항 직접 수정 → 재검증. 새 발견이 없을 때까지 반복(최대 2라운드). 위험 클래스(인증·결제·노출·시크릿) diff면 멀티렌즈 강제(cy6 참고).

종료: 테스트·빌드 그린 + 리뷰 통과 코드 + 변경 요약 출력 + 사이클 기록 파일(`docs/cycles/cycle_<세대>/<NN>-<기능슬러그>.md`, cy6에서 작성 — NN=직전 회차+1, 2단 인덱싱으로 작업순서 가시화). 커밋은 자동으로 하지 않는다(작업 브랜치 보존) — 마지막에 /commit 제안만. (연속/자동커밋은 사용자가 /cycle 인자로 명시했을 때만, 그리고 주기 체크포인트를 지킨다.)

제약(THESIS): 설계철학(결론 먼저·수치 비노출·"지금 사세요" 금지·과신방지 문구), 임계값은 설정값, any 금지, 시크릿 하드코딩 금지. 범위는 요청된 것만. 한국어.
