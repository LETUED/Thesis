★ v3 핵심 원칙 — 방향도 외부 오라클로 (LLM 자가판정은 신뢰 불가: Huang ICLR'24 arXiv:2310.01798, Kamoi TACL'24 arXiv:2406.01297):
가치·드리프트·산출 규모를 '한 줄로 대봐'식 자가서술로 판정하지 않는다. **git에서 측정 가능한 신호**(변경 파일·surface·LOC·회차 커밋)로 게이트한다. 이 게이트는 에이전트가 '통과하려고 최적화하는 목표'가 아니라 회차 종료 후 별도로 적용하는 **사후 감사**다 — 게이트를 목표로 노출하면 라벨 분식·우회가 생긴다(cycle_2 '신선도 4분할'이 그 예; obfuscated reward hacking: Baker OpenAI '25 arXiv:2503.11926).

★ 방향 게이트 (cy1 전에 git으로 자동 산출 — 자가申告 아님):
직전 회차 커밋을 git으로 분석한다.
- `git log --oneline --grep='cycle_3' -8` 로 최근 회차, 각 회차 `git show --stat <hash>` 로 변경 파일.
- **surface 도출**: 변경 파일 경로 → 영역(예: components/dashboard/*→`dashboard`, app/screener·components/screener/*→`screener`, backend/app/engine/*→`regime-engine`, lib/*→`frontend-lib`). 경로에서 기계적으로.
- **드리프트 차단**: 같은 surface가 직전 3회 연속이면 그 surface 금지 — 미접촉 surface에서 후보 ≥2 제시 후 택일.
- **정비 예산**: 직전 5회에 정비(테스트·기술부채·커버리지) 0회면 이번을 정비로 강제(유지보수는 무가치 아닌 안전성·신뢰성 가치 — MO-GRPO가 저분산 보상 회피를 reward hacking으로 규정). 정비 연속은 2회까지.
- **최소 산출 임계**: 직전 회차 순 프로덕션 변경이 '의미변화 없는 1줄/속성'이면 독립 회차로 인정 않고 이번에 인접 작업과 병합. value_size 'small' 3연속이면 더 큰 가치 후보 강제.

★ 주기 체크포인트 (연속/무한 모드):
- 사람이 방향을 직접 줄 수 있으면 N(기본5)회마다 AskUserQuestion.
- 사람이 '계속'으로 위임했어도 **체크포인트를 생략하지 않는다** — 대신 위 방향 게이트(git 자동 분석)를 강제 실행해 surface 편중이면 기계가 방향을 전환한다(사람 부재 fallback; AgentCollab류 self-eval은 자가판정 한계로 단독 신뢰 불가하므로 git 신호로 트리거).

방법 — 기계 게이트로 품질 보증(사람 승인 대체):
- 설계는 스스로 비판, 진행은 테스트·빌드 통과 강제, 리뷰는 독립 평가자 검증 후 즉시 수정.
- 시작 시 TaskCreate로 6단계 체크리스트, 단계마다 in_progress/completed 갱신(가시성용).
- 판단 갈리면 합리적 기본값 택해 decision log에 남기고 계속.

브라운필드 안전수칙(THESIS는 기존 코드베이스):
- 한 사이클 스코프 = 기능 1개. 큰 건 쪼개 여러 번.
- 파괴적·광범위 변경 예상이면 git worktree 격리(샌드박스 = 권한스킵의 유일한 보안경계).
- 사람은 루프 '밖'에서 스펙·리뷰만("sit on the loop, not in it").

단계 — 각 단계는 해당 cy 스킬을 Skill 도구로 실제 호출해 그 안내대로 수행:
1. 조사 cy1 → 2. 검색 cy2 → 3. 구성 cy3 → 4. 설계 cy4 → 5. 진행 cy5 → 6. 리뷰 cy6. 인자로 작업+직전 산출 전달. (방향 게이트는 cy1 직전 git 분석으로 1회, cy3에서 재확인.)

기계 게이트(사람 승인 대체):
- 설계(4) 후: 범위·제약·타입·엣지 자가점검 → 문제 있으면 고쳐 진행.
- 진행(5) 후: 테스트·빌드/타입체크 통과해야 다음. 실패 시 고쳐 재시도(최대 3회), 그래도면 로그 후 리뷰에서.
- 리뷰(6): 독립 평가자(fresh-context 서브에이전트) 검토 → 직접 수정 → 재검증, 새 발견 없을 때까지(최대 2R). 위험클래스(인증·결제·노출·시크릿) diff면 멀티렌즈 강제. 테스트 위조 감시(빈 assertion·하네스 우회) 포함(cy6).

★ 종료 (v3 — 회차 단위 커밋 강제):
테스트·빌드 그린 + 리뷰 통과 코드 + 회차 기록(`docs/cycles/cycle_3/<NN>-<슬러그>.md`, cy6 작성, git 자동도출 가치필드 포함) + **이번 회차를 단독 커밋**(`feat(cycle_3): NN-슬러그`, 본문에 surface·net LOC·value_size). squash·회차 묶음 금지 — 회차=커밋이라야 추적·롤백·자동 A/B 검증이 된다(v2 squash는 추적성 0이었음). 연속 모드면 다음 회차 cy1로 자동 재점화하되 방향 게이트(git 분석)를 먼저.
