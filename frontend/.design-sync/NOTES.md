# design-sync notes — thesis-frontend

This repo is a **Next.js app, not a component-library package** — there's no built `dist/` of components. The sync uses a hand-built synth path. Key gotchas:

## Build invariants (don't regress)
- **Always pass `--entry .design-sync/ds-entry.mjs`.** Without `--entry`, package-build sets `PKG_DIR = node_modules/thesis-frontend` (doesn't exist → ENOENT crash). With this entry, PKG_DIR walks up to the app root.
- **`ds-entry.mjs` is generated by `.ds-sync/discover-app.mjs`** (`export *` from every component file + a `componentSrcMap`). Its FIRST line is `import "./ds-process-shim.mjs"` — that shim sets `globalThis.process` so the browser IIFE doesn't throw `process is not defined` (lib/api.ts + supabase client read `process.env.NEXT_PUBLIC_*`). Drop it and the whole bundle dies → `[BUNDLE_EXPORT] 43/43 not a component`.
- **`componentSrcMap` in config.json is hand-maintained** (auto-discovery is off when `--entry` is set). When components are added/removed, re-run `node .ds-sync/discover-app.mjs` and merge its `componentSrcMap` output into config. `LabUiContext` (a React context, not a component) and the 5 `Card*` sub-parts + `BlockCanvas` are set to `null` (excluded from the card list; still importable via the bundle's `export *`).
- **tsconfig**: `cfg.tsconfig = .design-sync/tsconfig.build.json` maps `@/* → ../*` and stubs `next/link`, `next/navigation`, `next/image`, `next/headers`, `server-only` (`.design-sync/stubs/`) so app-coupled components don't crash previews.

## CSS (manual pre-step — converter does NOT run Tailwind)
Re-compile whenever components/tokens/`globals.css` change, BEFORE `package-build`:
```
./node_modules/.bin/tailwindcss -c ./.design-sync/tailwind.build.ts -i ./app/globals.css -o ./.design-sync/compiled.css
```
`tailwind.build.ts` adds a **safelist** so every semantic token utility (`bg-card`, `text-regime-on`, `bg-regime-off-muted`, `bg-surface-1`, …) is emitted — Tailwind otherwise only emits utilities the existing app used, and the design agent needs the full palette. `cfg.cssEntry = .design-sync/compiled.css` is appended to `_ds_bundle.css` and imported by `styles.css`.

## Render check
No playwright browser is installed; the render check drives **system Chrome**:
```
DS_CHROMIUM_PATH='C:\Program Files\Google\Chrome\Application\chrome.exe' node .ds-sync/package-validate.mjs ./ds-bundle
```
`playwright` (npm, no browsers) is installed under `.ds-sync/node_modules`.

## Re-sync risks / watch-list
- **Tailwind compile + discover-app are NOT run by the converter** — re-run both manually before `package-build` when source changes, or the bundle/CSS go stale.
- **App-coupled components ship floor cards.** Rich previews for `lab/*` (BlockCanvas, MarketDrawer, CompanySearch, BlockLab — React Flow + Supabase + API), `billing/*`, `LogoutButton`, `SessionWatch`, `Sidebar`, `AppShell`, and data-driven `dashboard/*` (RegimeSignalCard, AllocationDonut/Panel, IndicatorsBoard, KoreaMacroBoard) need realistic prop data (RegimeResult/AllocationResult shapes from `lib/types`) and sometimes providers/mocks. Author them in `.design-sync/previews/<Name>.tsx`, then re-build + re-capture + re-grade.
- **Not a git repo.** Durable inputs (`.design-sync/config.json`, `conventions.md`, `NOTES.md`, `previews/`, `stubs/`, `tsconfig.build.json`, `ds-entry.mjs`, `ds-process-shim.mjs`, `tailwind.build.ts`) are NOT committed. `git init` + commit them (gitignore `.ds-sync/`, `ds-bundle/`, `.design-sync/.cache/`, `.design-sync/node_modules`) for reproducible re-syncs.
- Re-sync upload path: this project is now non-empty → the next sync takes the **atomic path** (`_ds_sync.json` anchor lets it skip unchanged components).
